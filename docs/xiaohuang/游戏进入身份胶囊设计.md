# 游戏进入身份胶囊设计

更新时间：2026-06-10

## 背景

当前 App 已经完成了大厅到游戏页的统一场景转场：大厅退出、游戏页进入、系统返回反向退场。这个基础动效解决了“切换太硬”的问题，但用户从大厅卡片进入游戏时，仍然缺少一个轻量的上下文承接。

身份胶囊是一个低风险过渡方案：进入游戏时在顶部短暂展示“图标 + 游戏名”，让用户明确刚进入的是哪个游戏。它类似轻量灵动岛提示，但不承担通知、操作入口或业务状态展示。

## 目标

- 给“点击大厅卡片进入游戏”增加轻量上下文承接。
- 不改任何游戏内部布局、规则、计分和主循环。
- 不拦截游戏触摸操作。
- 不引入真正 `geometryTransition` 的结构风险。
- 代码按壳层功能拆分，避免继续膨胀 `Index.ets`。

## 效果

进入游戏时：

```text
大厅卡片点击
  -> 游戏页入场
  -> 顶部出现身份胶囊
  -> 短暂停留
  -> 自动淡出
```

胶囊内容：

- 游戏图标。
- 游戏名称。
- 半透明暖色背景。
- 轻微阴影。

动效：

- 透明度 `0 -> 1 -> 0`。
- 纵向位置 `-8vp -> 0 -> -8vp`。
- 单段动画时长 `180ms`。
- 停留时间 `820ms`。

## 文件职责

```text
entry/src/main/ets/pages/Index.ets
```

负责：

- 判断什么时候进入游戏。
- 维护 `entryBadgeGameId`、`entryBadgeMounted`、`entryBadgeVisible`。
- 控制显示和隐藏定时器。
- 返回大厅时主动隐藏胶囊。

不负责：

- 胶囊 UI 样式。
- 胶囊内部布局。

```text
entry/src/main/ets/shell/GameEntryBadge.ets
```

负责：

- 根据 `gameId` 读取游戏注册信息。
- 渲染游戏图标、标题和颜色。
- 执行透明度和位移动效。
- 使用 `HitTestMode.None`，确保不拦截游戏操作。

```text
entry/src/main/ets/design/CoopMotion.ets
```

负责：

- `entryBadgeDuration`
- `entryBadgeHoldDuration`
- `entryBadgeOffsetVp`

## 为什么放在 shell

身份胶囊不是普通通用组件，它只服务“从大厅进入游戏”的壳层场景，并依赖 `GameRegistry` 中的游戏元信息。因此放在 `shell/` 比放在 `components/` 更合适。

`components/` 保留给真正跨页面、跨场景的通用组件，例如按压反馈、弹层转场。

## 当前不做

- 不做真正 `geometryTransition`。
- 不把大厅卡片位置传给游戏页。
- 不改变游戏页顶部布局。
- 不在返回大厅时显示胶囊。
- 不把胶囊做成可点击入口。
- 不接入真实震动反馈。

## 后续升级方向

如果当前效果保留，可以考虑两个方向：

1. 继续低风险优化：调整位置、停留时长、透明度和圆角。
2. 做真正共享元素试点：从大厅卡片 icon/accent 色块过渡到游戏页身份区域。

真正共享元素要等游戏页结构更稳定后再做，因为它需要 in/out 组件配对，且容易和现有场景转场抢视觉焦点。

## 验收点

- 点击任意游戏，胶囊显示的图标和游戏名正确。
- 胶囊出现和游戏页入场节奏自然。
- 胶囊自动淡出，无残留。
- 胶囊不影响顶部按钮和游戏操作。
- 快速返回大厅时胶囊会隐藏。
- 连续进入不同游戏时，显示最新游戏信息。

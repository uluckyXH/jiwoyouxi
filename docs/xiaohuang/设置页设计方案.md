# 设置页设计方案

更新时间：2026-07-24

## 背景

当前设置页位于大厅的第四个 Tab，目前只应提供已经确认的深浅色模式。此前页面中的 iOS / Android 说明属于开发阶段备注，不应出现在正式 App 界面。

本轮目标不是增加更多设置项，而是把已有的真实控制整理成一个安静、清晰、和大厅视觉一致的设置页。

## 目标

- 移除 iOS / Android 和开发阶段相关的说明文案。
- 将深浅色从二元开关改为明确的模式选择。
- 不展示声音、触感反馈等尚未确认的设置项。
- 与首页、收藏页共用暖色/深色主题、细边框和克制阴影。
- 不加入没有实际功能的账号、通知、版本、隐私或平台入口。

## 页面定位

设置页是轻量偏好面板，不是“更多功能”列表，也不是关于页。用户进入这里通常只会做一次明确调整，因此页面应保持低密度、低干扰、容易扫视。

标题继续使用现有的“设置”样式和页面顶部节奏，不增加 Hero、说明横幅或大段辅助文字。

## 信息结构

```text
设置（使用底部设置 Tab 同款图标）

外观
  [ 太阳 浅色 | 月亮 深色 ]
```

仅保留一个分组：

- `外观`：主题模式，是本页唯一且最主要的设置。

不保留平台说明、功能路线图或静态“关于”文字。

## 视觉方案

### 页面基础

- 浅色模式沿用大厅背景 `#fff4e4`。
- 深色模式沿用大厅背景 `#101110`。
- 内容使用分组面板，而非每一行单独做成厚重卡片。
- 分组面板使用现有首页的暖白/深暖灰底、细暖色边线和轻阴影；圆角保持克制，避免层层嵌套的卡片感。

### 外观组

外观组是一块独立面板，内部使用两个等宽的分段选项：

- 浅色：太阳图标 + “浅色”。
- 深色：月亮图标 + “深色”。

选中项使用主题强调色和清晰图标状态，未选中项保持安静的面板底色。两个选项具有相同点击面积，切换后立即更新大厅主题并保留现有淡入淡出动画。

不再显示“深色模式：开/关”，因为“浅色 / 深色”是模式选择，不是单纯的布尔功能。

### 间距与尺寸

- 标题与第一组之间保留现有页面节奏。
- 分组标题使用较小的次级文字，避免和页面标题竞争。
- 主题分段控件高度为 `58vp`。
- 设置标题使用 `sys.symbol.more`，与底部设置 Tab 复用同一系统图标资源。

## 浅色与深色细节

浅色模式：

- 面板使用暖白，边框使用低透明暖棕。
- 选中的主题段使用品牌金，文字保持高对比。

深色模式：

- 面板使用深暖灰，与大厅卡片保持一致。
- 未选中分段保持低对比但文字仍可读。
- 选中主题段使用更亮的暖金。

## 交互与状态

### 外观

- 点击“浅色”或“深色”后立即切换主题。
- 点击当前已选模式不重复写入状态。
- 沿用 `appearanceMode` 的本地持久化，无需新增数据结构。
- 首页头部的日/月快捷切换入口继续保留，并与设置页状态同步。

## 实现范围

主要修改 `entry/src/main/ets/pages/HubPage.ets`：

- 重做 `Settings()` 的分组结构。
- 用 `AppearanceModeSelector()` 替代“深色模式”文本开关。
- 用 `SettingsTitleRow()` 复用底部设置 Tab 的系统图标。
- 删除 iOS / Android 说明文本。
- 复用现有深浅色 palette helper、动画和状态回调。

不需要修改：

- `CoopAppSnapshot`、`CoopStorage` 的主题持久化结构。
- `Index.ets` 的主题更新接口。
- 任意游戏内部页面或游戏设置。

## 验收点

- 设置页不再出现 iOS、Android 或开发阶段说明。
- 页面只展示外观模式这一项已确认的控制能力。
- 深浅色使用“浅色 / 深色”分段选择，状态明确且可立即生效。
- 设置标题图标与底部设置 Tab 使用同一系统图标资源。
- 浅色、深色模式下文字、图标、边框和选中状态均清晰可读。
- 切换主题后首页、收藏、成就、设置和底部 Tab 状态保持同步。
- 小屏和手机画布模式下没有文字截断、控件重叠或底部 Tab 遮挡。

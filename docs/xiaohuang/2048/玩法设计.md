# 鸡窝 2048 玩法设计

## 2048 是什么

2048 是一个 4x4 方格滑动合成游戏。

玩家每次向上、下、左、右滑动，棋盘上的所有方块会朝同一个方向移动。相同数字的方块撞在一起会合成更大的数字，例如 `2 + 2 = 4`、`4 + 4 = 8`。每次有效移动后，棋盘会随机新增一个小方块。目标是在棋盘被填满之前合成 `2048`。

它适合放进《鸡窝游戏》，原因是：

- 规则简单，离线可玩。
- 单局节奏轻，适合随手打开。
- 4x4 棋盘和合成反馈很容易做成软萌玩具质感。
- 数字成长清楚直观，适合做成暖色、圆润、轻松的棋盘体验。

## 游戏定位

游戏名建议：

```text
鸡窝 2048
```

大厅标签：

```text
数字叠叠乐
```

一句话描述：

```text
滑动数字块合成升级，冲到 2048。
```

整体方向是“暖色玩具棋盘里的 2048”。保留《鸡窝游戏》的圆润、轻松、离线休闲气质，但本版本不使用旧版主题包装。推荐做法是：

- 数字仍然保留，保证 2048 玩家一眼能玩。
- 每个等级优先用颜色和数字大小区分，不额外堆主题图标。
- 棋盘像一个圆角木质收纳棋盘。
- 合成时用软弹缩放和少量圆点反馈，不做复杂装饰。

## 核心玩法

### 棋盘

- 固定 4x4。
- 每个格子是一个圆角棋盘凹槽。
- 初始随机生成 2 个低等级数字块。
- 新数字块只在空格中生成。

### 操作

- 支持上下左右滑动。
- 一次滑动内，每个数字块最多合成一次。
- 无效滑动不新增数字块。
- 结算、暂停、弹窗期间不响应棋盘滑动。

### 生成规则

每次有效移动后生成一个新数字块：

- `90%` 概率生成 `2`。
- `10%` 概率生成 `4`。

### 合成规则

- 相同数值才能合成。
- 合成后数值翻倍。
- 一次移动中，已经合成过的数字块不能继续合成。
- 首次合成 `2048` 时触发成功达成弹窗。
- 达成后允许玩家选择“继续挑战”。

### 失败规则

当同时满足下面两个条件时失败：

- 棋盘没有空格。
- 任意相邻的上下左右格子都没有相同数值。

失败文案：

```text
棋盘挤满了！
```

## 等级包装

当前版本不做蛋元素等级包装，内部和外显都以标准 2048 数字为主。

| 数值 | 展示重点 | 视觉方向 |
| --- | --- | --- |
| 2 | 入门数字块 | 奶油白，最轻的视觉重量 |
| 4 | 入门数字块 | 浅黄，和 `2` 明确区分 |
| 8 | 暖色数字块 | 浅粉，数字保持高可读 |
| 16 | 清爽数字块 | 浅青绿，打破单一暖色 |
| 32 | 暖橙数字块 | 橙色，合成反馈更明显 |
| 64 | 木色数字块 | 奶油棕，承接棋盘边框 |
| 128 | 高等级数字块 | 浅蓝，字号略收紧 |
| 256 | 高等级数字块 | 珊瑚粉，强调进度 |
| 512 | 高等级数字块 | 草绿，和低等级拉开 |
| 1024 | 冲刺数字块 | 暖金，数字优先 |
| 2048 | 目标数字块 | 亮黄，触发成功达成弹窗 |
| 4096+ | 继续模式数字块 | 紫色或特殊色，不作为首版目标 |

设计原则：

- 数字字号要足够大，不能被装饰遮挡。
- 颜色服务于识别，不追求强主题贴纸。
- `2048` 的目标感来自颜色、弹窗和合成反馈，不再依赖蛋或角色贴图。

## 视觉风格

### 场景概念

画面像一只鸡把家里的小木桌改成了 2048 棋盘。

建议构图：

- 顶部是小木牌标题：`鸡窝 2048`。
- 标题旁边有半眯眼主角鸡头像。
- 中部是 4x4 木质圆角棋盘。
- 棋盘下面有两个小按钮：`再来一局`、`回大厅`。
- 背景保留鸡窝小屋：木地板、圆形墙、小旗子、软萌小道具。

### 色彩

沿用《鸡窝游戏》整体色系：

- 奶油白：页面底色和低等级数字块。
- 亮黄：按钮、高亮、低等级合成。
- 粉衬衫色：品牌强调色和关键等级。
- 深棕色：文字、描边、棋盘木框。
- 浅草绿 / 浅青蓝：高等级辅助色，避免整屏只有粉黄奶油。

注意：

- 不使用刺眼霓虹色。
- 不使用纯黑大字。
- 棋盘和数字块都要有深棕描边，但描边不能压过数字识别。

### 棋盘

棋盘像一个木质收纳棋盘：

- 外框使用奶油棕或木色。
- 4x4 凹槽略深，方便看清格位。
- 格子间距稳定，不能因为合成动画产生布局抖动。
- 圆角可比普通 2048 更大，但仍保持方块识别。

### 数字块

数字块像软糖玩具牌：

- 圆角矩形。
- 微 3D 高光。
- 深棕描边。
- 轻微投影。
- 中间显示大数字。

低等级可偏奶油、淡黄、粉色；高等级逐渐加入青蓝、草绿、珊瑚红，让棋盘有成长感。

## 动效

动效关键词：弹、啵、咚、飘。

### 滑动

- 数字块滑动时间控制在 `100ms` 到 `140ms`。
- 滑动路径要清楚，避免瞬移。
- 快速连续滑动时要么排队，要么忽略动画中的输入，不能状态错乱。

### 新增数字块

- 新数字块轻微弹一下。
- 伴随轻微 `啵` 的反馈。
- 不需要大粒子，避免干扰下一步判断。

### 合成

- 合成后的数字块先放大，再回到 `1.0`。
- 数字短暂跳动。
- 出现少量卡片内圆点。
- 合成 `512` 以上时可以加更明显的光圈。

### 达成 2048

- 棋盘轻微震动一次。
- `2048` 数字块弹一下。
- 出现统一成功达成卡片。
- 不要长时间阻断操作，玩家选择继续后立即回到棋盘。

## 音效方向

首版可以先沿用现有通用点击音，后续补专属音效：

- 滑动：轻轻的木块摩擦声。
- 合成：软弹 `啵`。
- 高等级合成：更亮的小铃声。
- 失败：轻微 `咚`，不要刺耳。
- 达成：短促庆祝，不要太长。

## 计分和存档

### 计分

使用标准 2048 分数规则：

- 每次合成时，增加合成后的数值。
- 例如 `32 + 32 = 64`，本局分数加 `64`。

这样玩家熟悉、可预期，也方便对照其他 2048 实现。

### 最高纪录

建议记录两类数据：

- `bestScore`：最高分。
- `bestTile`：最大数字。

大厅卡片可以优先展示：

```text
最大数字 2048
```

如果还没玩过：

```text
等你挑战
```

## 首版实现范围

首版只做稳定可玩闭环：

- 4x4 标准 2048 规则。
- 上下左右滑动。
- 数字合成和分数。
- 最高分保存。
- 成功达成、失败结算、重开、回大厅。
- 鸡窝主题棋盘和数字块视觉。
- 新增、滑动、合成三个基础动效。

### 开发调试彩蛋

为了方便真机测试成功/失败弹窗，2048 页面保留隐藏开发入口：

- 连续点击页面标题 `鸡窝 2048` 五次，显示开发测试面板。
- `触发2048`：直接进入成功达成弹窗，用来检查成功卡片、按钮和回大厅记录。
- `触发失败`：直接进入失败结算弹窗，用来检查失败卡片、重开和回大厅。
- `隐藏`：收起开发测试面板。

该面板默认不显示，不参与正式玩法，不影响正常滑动和方向键操作。

首版暂不做：

- 道具。
- 撤销。
- 多棋盘尺寸。
- 排行榜。
- 复杂成就。
- AI 提示。

## 验收清单

- 开局生成 2 个数字块。
- 有效滑动后才新增数字块。
- 同一数字块单次滑动最多合成一次。
- `2 -> 4 -> 8 -> ... -> 2048` 合成链正确。
- 分数等于每次合成后的数值累加。
- 首次出现 `2048` 时弹出成功达成弹窗。
- 继续游戏后不重复弹同一次 2048 达成提示。
- 棋盘无空格且无可合成相邻格时失败。
- 重开后棋盘、分数、达成状态重置。
- 返回大厅不丢最高纪录。
- 小屏手机上标题、分数、棋盘、按钮不重叠。
- 快速连续滑动不会产生重复合成或丢块。

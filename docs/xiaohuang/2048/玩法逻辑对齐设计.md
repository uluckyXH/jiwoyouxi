# 鸡窝 2048 玩法逻辑对齐设计

## 目标

当前 2048 首版实现已经能进入页面，但交互和逻辑表现还没有达到可验收状态。

本阶段先暂停继续堆视觉和特效，目标改为：

- 对齐原版 2048 的核心规则。
- 把滑动、方向键、动画状态统一到同一套逻辑。
- 先删掉干扰判断的复杂粒子和贴纸特效。
- 只保留三类基础动画：移动、合并放大、新块出现。
- 做到“能玩、逻辑对、反馈清楚”，再进入素材和主题化阶段。

## 原版 2048 核心逻辑

### 初始化

- 棋盘固定 `4 x 4`。
- 开局生成 2 个初始方块。
- 每个初始方块随机出现在空格。
- 新方块数值：
  - `90%` 概率是 `2`。
  - `10%` 概率是 `4`。

### 输入方向

原版方向编码：

```text
0: up
1: right
2: down
3: left
```

手机端通过滑动生成方向，键盘端通过方向键生成方向。

我们在鸿蒙版本中采用：

- 主输入：滑动棋盘。
- 辅助输入：页面下方方向键。
- 两者必须调用同一个 `move(direction)` 入口。
- 不能让按钮逻辑和滑动逻辑各走一套规则。

### 移动顺序

原版不是简单把数组压缩一下，而是按方向决定遍历顺序。

原则：

- 向右移动时，从右往左处理。
- 向左移动时，从左往右处理。
- 向下移动时，从下往上处理。
- 向上移动时，从上往下处理。

原因是每个块都要先找到自己在当前方向上能到达的最远位置，再判断前方是否可以合并。

### 单块移动

每个方块移动时：

1. 先沿方向找最远空格。
2. 再看最远空格前方一格是否存在同值方块。
3. 如果前方同值且该方块本轮还没合并过，则合并。
4. 否则移动到最远空格。

### 合并限制

每个方块一轮移动只能参与一次合并。

典型例子：

```text
[2, 2, 2, 0] 向左 => [4, 2, 0, 0]
[2, 2, 2, 2] 向左 => [4, 4, 0, 0]
[4, 4, 8, 0] 向左 => [8, 8, 0, 0]
```

不能出现：

```text
[2, 2, 2, 0] 向左 => [4, 0, 0, 0]
```

也不能出现一次移动里新合成的 `4` 继续和后面的 `4` 合并。

### 新方块生成

只有本次移动真的改变棋盘时，才生成新方块。

无效移动不做任何事：

- 不加分。
- 不生成新块。
- 不播放移动动画。
- 不改变棋盘状态。

### 分数

每次合并后，分数增加合并后的数值。

例子：

```text
2 + 2 = 4，加 4 分
4 + 4 = 8，加 8 分
```

一轮移动有多个合并时，累加所有合并值。

### 胜利

首次出现 `2048` 时进入胜利状态。

胜利弹窗应该提供：

```text
继续挑战
回鸡窝
```

选择继续后：

- 关闭胜利弹窗。
- 允许继续合成 `4096+`。
- 同一次 `2048` 不重复弹窗。

### 失败

失败条件：

- 棋盘没有空格。
- 上下左右任意相邻格都没有相同数值。

只有这两个条件同时满足，才结束。

## 当前实现需要修正的问题

### 1. 视觉特效干扰判断

当前版本有粒子、角标、贴纸式标记等视觉内容，容易干扰判断“方块到底有没有移动/合并”。

下一版先删除：

- 合成粒子。
- 飘动符号。
- 方块角标文字。
- 高等级额外光效。

保留：

- 数字。
- 方块底色。
- 方块移动。
- 合并放大。
- 新块出现。

### 2. 方向键遮挡

方向键需要保留，但必须满足：

- 箭头完整可见。
- 按钮尺寸固定。
- 文本居中。
- 不遮挡棋盘。
- 不挤压底部安全区。
- 禁用状态可以变淡，但箭头仍然可读。

建议样式：

```text
       ↑
   ←   ↓   →
```

按钮大小建议：

- 小屏：`48 x 48`
- 常规屏：`54 x 54`
- 圆形或轻圆角都可以。

### 3. 动画状态不应该改变规则结果

逻辑状态和动画状态要分离。

规则层输出：

- 本次是否有效移动。
- 最终棋盘。
- 每个方块的起点。
- 每个方块的终点。
- 合并事件。
- 新生成方块。
- 分数增量。

页面层只消费这些结果播放动画，不能在动画过程中重新推导规则。

## 推荐实现模型

### 数据结构

建议把方块拆成逻辑字段和动画字段：

```ts
interface Tile {
  id: number;
  value: number;
  row: number;
  col: number;
  previousRow: number;
  previousCol: number;
  mergedFromIds: number[];
  isNew: boolean;
  isMerged: boolean;
}
```

说明：

- `row/col` 是最终位置。
- `previousRow/previousCol` 是动画起点。
- `mergedFromIds` 用于记录两个旧块合成一个新块。
- `isNew` 控制新块出现动画。
- `isMerged` 控制合并放大动画。

### 移动返回结果

规则函数建议返回：

```ts
interface MoveResult {
  moved: boolean;
  scoreGain: number;
  won: boolean;
  over: boolean;
  beforeTiles: Tile[];
  afterTiles: Tile[];
  spawnedTile: Tile | null;
  mergeEvents: MergeEvent[];
}
```

页面播放顺序：

1. 使用 `beforeTiles` 渲染当前状态。
2. 用户输入方向。
3. 规则层计算 `MoveResult`。
4. 如果 `moved === false`，直接结束。
5. 播放移动动画：从 `previousRow/previousCol` 到 `row/col`。
6. 移动结束后显示合并结果，并给 `isMerged` 方块做放大动画。
7. 再显示 `spawnedTile`，做新块出现动画。
8. 检查胜利/失败弹窗。

### 动画约束

只做这三种动画：

| 动画 | 时长 | 说明 |
| --- | --- | --- |
| 移动 | `90ms - 120ms` | `translate` 到目标槽位 |
| 合并放大 | `120ms - 160ms` | `scale 1 -> 1.12 -> 1` |
| 新块出现 | `120ms - 180ms` | `scale 0 -> 1` 或 `0.72 -> 1` |

暂不做：

- 粒子。
- 羽毛。
- 星星。
- 屏幕震动。
- 大面积阴影动画。
- 复杂贴纸切换。

## 交互规范

### 滑动

- 滑动区域就是棋盘。
- 单指滑动。
- 取 `dx/dy` 中绝对值更大的方向。
- 阈值建议 `10px - 16px`，不要太大。
- 动画播放中锁输入。
- 无效滑动不锁输入、不生成新块。

### 方向键

- 方向键调用同一个 `move(direction)`。
- 点击时可以有轻微按压反馈。
- 方向键不能承担不同规则。
- 方向键不是调试按钮，是正式辅助操作。

### 快速连续输入

首版建议简单处理：

- 动画中忽略新的输入。

后续再考虑输入队列：

- 动画期间最多缓存 1 次下一方向。
- 动画结束后立即执行缓存方向。

## 验收用例

### 移动与合并

```text
[2, 2, 0, 0] left => [4, 0, 0, 0]
[2, 0, 2, 0] left => [4, 0, 0, 0]
[2, 2, 2, 0] left => [4, 2, 0, 0]
[2, 2, 2, 2] left => [4, 4, 0, 0]
[4, 4, 8, 8] right => [0, 0, 8, 16]
```

### 无效移动

```text
[2, 4, 8, 16] left
```

如果这一行已经在最左，且没有其它行变化：

- 不生成新块。
- 不加分。
- 不播放移动动画。

### 胜利

```text
[1024, 1024, 0, 0] left => [2048, 0, 0, 0]
```

首次出现 `2048` 弹胜利。

### 失败

满盘棋盘：

```text
2 4 2 4
4 2 4 2
2 4 2 4
4 2 4 2
```

没有相邻同值，判定失败。

# 扫雷结算弹窗与羽毛动效设计

## 目标

扫雷结算从当前的纯 ArkUI 卡片升级为鸡窝主题弹窗。

失败时表现为踩到鸡雷后鸡毛短时飞散，随后出现失败弹窗。胜利时不做爆炸或强刺激效果，直接出现成功弹窗，可以配少量轻快羽毛或星光感装饰。

## 可用素材

资源目录：

```text
entry/src/main/resources/rawfile/games/minesweeper/
```

当前可用素材：

- `popup_failure_card.png`：失败弹窗底图，尺寸 `1259x970`。
- `popup_success_card.png`：成功弹窗底图，尺寸 `1132x1004`。
- `feather_01.png` 到 `feather_10.png`：羽毛飘落素材，尺寸不完全一致，适合作为随机羽毛粒子。
- `button_yellow.png`：主要操作按钮，适合“再来一局”。
- `button_green.png`：次要操作按钮，适合“回大厅”或“继续查看”。
- `button_wood_small_right.png` / `button_wood_small_left.png`：小按钮底图，可用于关闭按钮。

## 失败流程

触发条件：

- 玩家在排雷模式点到鸡雷。

流程：

1. 踩中的格子统一显示 `tile_mine.png` 鸡雷图，避免玩家误以为存在“碎砖雷”等第二种危险物。
2. 棋盘其他鸡雷按现有逻辑翻开。
3. 立即播放羽毛飞散和飘落，时长建议 `900ms` 到 `1400ms`。
4. 羽毛动效开始后 `250ms` 到 `350ms` 出现失败弹窗。
5. 弹窗文案显示“踩到鸡雷了！”。

失败弹窗使用：

```text
popup_failure_card.png
```

失败弹窗语气：

- 主标题：`踩到鸡雷了！`
- 副文案：`鸡窝里混进了鸡雷，再试一次。`
- 数据行：`已排查 23/54`

## 胜利流程

触发条件：

- 所有安全格被翻开。

流程：

1. 自动给所有鸡雷格插旗，保持当前胜利逻辑。
2. 弹出成功弹窗。
3. 可选播放少量羽毛轻飘效果，不做密集飞散。

成功弹窗使用：

```text
popup_success_card.png
```

成功弹窗语气：

- 主标题：`鸡窝安全！`
- 副文案：`所有鸡雷都被你避开了。`
- 数据行：`用时 00:38 · 得分 9620 · 最高 9620`

## 弹窗布局

弹窗容器使用 `Stack`：

1. 半透明遮罩铺满屏幕。
2. 羽毛层位于遮罩上方、弹窗下方。
3. 弹窗卡片图位于中间。
4. 文本和按钮叠在卡片空白区域。
5. 关闭按钮位于弹窗右上角，略微压在卡片边缘上。

建议尺寸：

- 弹窗宽度：`Math.min(pageWidth * 0.86, 390)`。
- 小屏宽度：`Math.min(pageWidth - 28, 360)`。
- 卡片使用原始比例缩放，不强行拉伸变形。
- 文案区域放在卡片下半部分的大空白区域。

按钮建议：

- 主按钮：`再来一局`，使用 `button_yellow.png`。
- 次按钮：`回大厅`，使用 `button_green.png`。
- 关闭按钮：新增一个小的 `X` 文本按钮，底图可以先用 `button_wood_small_right.png`，后续如果有关闭图标再替换。

关闭按钮行为：

- 只关闭弹窗，不重置棋盘。
- 关闭后游戏仍保持结算态，玩家可以查看棋盘结果。
- 页面保留一个入口重新打开结果弹窗，或者允许点击“重新开始”直接开新局。

如果实现成本要更低，第一版关闭按钮可以等同于“回大厅”。但推荐做成“仅关闭弹窗”，这样玩家能看到最后踩中的格子和鸡雷分布。

## 羽毛动效设计

失败羽毛：

- 数量：`12` 到 `18` 个。
- 素材：从 `feather_01.png` 到 `feather_10.png` 随机选。
- 初始位置：以踩雷格附近为主，少量从屏幕上半部分随机落下。
- 横向位移：`-80vp` 到 `80vp`。
- 纵向位移：`80vp` 到 `220vp`。
- 旋转：`-35deg` 到 `35deg`。
- 透明度：从 `1` 渐隐到 `0`。
- 时长：`900ms` 到 `1400ms`。

胜利羽毛：

- 数量：`4` 到 `8` 个。
- 动作更慢，透明度更低。
- 不遮挡主标题和按钮。

性能约束：

- 羽毛只在结算开始时播放一次。
- 羽毛层播放结束后从组件树移除，避免常驻动画。
- 不在每帧重新生成随机数，进入结算时一次性生成羽毛配置。

## 代码落点

主要改动文件：

```text
entry/src/main/ets/games/minesweeper/MinesweeperResultOverlay.ets
```

建议新增或拆分：

```text
MinesweeperFeatherLayer.ets
MinesweeperResultButtons.ets
```

页面状态建议：

- `showResultOverlay: boolean`：控制弹窗是否显示。
- `resultOverlayDismissed: boolean`：区分游戏已结束但弹窗被关闭。
- `featherSeed` 或 `featherItems`：进入结算时生成羽毛轨迹。

`MinesweeperPage.ets` 负责：

- 游戏结束时打开弹窗。
- 传入胜负、用时、得分、最高分。
- 处理“再来一局”“回大厅”“关闭弹窗”。

`MinesweeperResultOverlay.ets` 负责：

- 根据 `win` 切换 `popup_success_card.png` / `popup_failure_card.png`。
- 渲染标题、数据、按钮。
- 提供关闭按钮。

## 第一版实现范围

第一版优先完成：

- 成功/失败弹窗底图接入。
- ArkUI 文案叠加。
- `再来一局`、`回大厅`、`关闭` 三个操作。
- 失败时显示羽毛飘落层。

第一版暂不做：

- 精确从踩雷格坐标发散。
- 复杂物理飘落。
- 声音效果。

## 验收点

- 失败时出现 `popup_failure_card.png`，标题为“踩到鸡雷了！”。
- 胜利时出现 `popup_success_card.png`，标题为“鸡窝安全！”。
- “再来一局”能重置并开始新棋盘。
- “回大厅”能返回大厅。
- “关闭”只关闭弹窗，不清空当前棋盘结果。
- 羽毛不会遮挡按钮超过一秒。
- 小屏下弹窗不顶到状态栏，不遮挡系统手势条。
- 构建命令通过：

```text
./scripts/build-app.sh --stacktrace
```

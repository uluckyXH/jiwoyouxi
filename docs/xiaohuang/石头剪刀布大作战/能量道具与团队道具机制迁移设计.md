# 石头剪刀布大作战能量道具与团队道具机制迁移设计

日期：2026-07-21
状态：第一版已实现，待真机验收
适用范围：新版 `entry/src/main/ets/games/rpsBattle/`，优先适配 `团队占点` 模式

## 结论

`能量道具` 和 `团队道具` 在原版里不是两套独立刷新的玩法。

- `能量道具` 对应 `powerUps`，负责在场上刷新可拾取道具。
- `团队道具` 对应 `teamPowerUps`，负责把拾取效果从“单个单位生效”改成“同阵营全队生效”。
- 因此 `teamPowerUps` 依赖 `powerUps`：没有能量道具，就没有团队道具效果。
- 原版 `团队占点` 预设默认同时开启 `controlZones`、`blackHole`、`lastStand`、`powerUps`、`teamPowerUps`。

新版迁移口径：

- 先只把这两个机制作为 `团队占点` 的增强机制迁移，不做全模式自由组合。
- `能量道具` 默认开启，并允许开局前关闭。
- `团队道具` 默认开启，并允许开局前关闭，但必须依赖 `能量道具`。
- 关闭 `能量道具` 时，`团队道具` 自动关闭。
- 打开 `团队道具` 时，自动打开 `能量道具`。
- 当前不改 `经典乱斗`、`小心叛徒`、`众生平等` 的运行时。

## 实施记录

2026-07-21 已完成第一版原生接入：

- 新增 `RpsBattlePowerUpSystem.ets`，独立处理道具刷新、过期、拾取、拾取脉冲和机制快照。
- `团队占点` 的 `powerUps` / `teamPowerUps` 从 `upcoming` 移到 `enabled`，默认开启，开局前可关闭。
- 引擎按“刷新/过期 -> 移动 -> 拾取并应用效果 -> 黑洞 -> 碰撞 -> 据点 -> 绝地求生 -> 结算”的顺序接入。
- 实体增加 `shield` 和 `speedBoostUntilSec` 状态；护盾在碰撞转阵前消耗，加速影响移动速度上限。
- 分裂走统一实体生成入口，并受全局人数上限保护；团队分裂每次拾取最多额外生成 8 个单位。
- Canvas 增加道具本体、拾取脉冲、护盾外圈和加速尾迹，不引入每帧 ArkUI 组件。
- `RpsBattlePerf` 增加 `powerUpMs`、`powerUpsActive`、`powerUpsPicked/frame`、`teamPowerUps/frame`。
- 新增 HarmonyOS Hypium 测试文件 `entry/src/ohosTest/ets/test/RpsBattlePowerUpSystem.test.ets`。

2026-07-21 对抗节奏调参：

- 首个道具提前到开局约 `3s`。
- 后续道具间隔调整为约 `4s`，随机浮动 `-0.8s..+1s`。
- 场上最多道具从 `4` 个提高到 `6` 个。
- 道具类型按 `加速 -> 护盾 -> 分裂` 循环刷新，避免真机测试时长时间只看到加速。
- 道具寿命仍为 `12s`，团队分裂仍受总人数上限和单次团队生成上限保护。

## 原版数据结构

原版道具类型：

| 枚举 | 名称 | 表现 |
| --- | --- | --- |
| `PowerUpKind.Speed` | 加速 | 提升单位速度一段时间 |
| `PowerUpKind.Shield` | 护盾 | 抵挡一次转化 |
| `PowerUpKind.Split` | 分裂 | 额外生成一个同阵营单位 |

原版 `PowerUp` 字段：

| 字段 | 说明 |
| --- | --- |
| `id` | 道具 ID，复用实体自增 ID |
| `kind` | 道具类型，三选一随机 |
| `x/y` | 场上坐标 |
| `radius` | 拾取和绘制半径，原版固定 `14` |
| `expiresAt` | 过期时间 |

原版实体里已有与道具相关的字段：

| 字段 | 说明 |
| --- | --- |
| `shield` | 护盾层数，原版道具设置为至少 `1` |
| `rageUntil` | 加速持续到的时间 |

新版 `RpsBattleEntity` 已补齐 `shield` 和 `speedBoostUntilSec`，道具效果不放在 UI 层或临时变量里。

## 原版刷新规则

原版常量：

| 参数 | 原版值 |
| --- | --- |
| 首次刷新 | 开局后约 `5000ms` |
| 基础刷新间隔 | `6500ms` |
| 间隔随机偏移 | `-1800ms` 到 `+2400ms` |
| 道具寿命 | `12000ms` |
| 场上最多数量 | `4` |
| 道具半径 | `14` |
| 出生范围 | 战场边界内缩 `34` |

原版 `updatePowerUps(now)` 行为：

1. 如果 `powerUps` 关闭，清空场上道具并返回。
2. 清理已经过期的道具。
3. 如果未到下一次刷新时间，返回。
4. 如果场上已有 `4` 个道具，返回。
5. 随机生成一个 `Speed / Shield / Split` 道具。
6. 设置下一次刷新时间。
7. 追加事件 `能量道具刷新`。

## 新版当前调参

这部分是新版为了提高团队占点对抗密度做的体验调参，不改写上面的原版记录。

| 参数 | 新版当前值 |
| --- | --- |
| 首次刷新 | 开局后约 `3s` |
| 基础刷新间隔 | `4s` |
| 间隔随机偏移 | `-0.8s` 到 `+1s` |
| 道具寿命 | `12s` |
| 场上最多数量 | `6` |
| 道具类型 | `加速 -> 护盾 -> 分裂` 循环 |

设计原因：

- 提高刷新速度，让玩家更容易看到护盾和分裂。
- 类型循环比完全随机更适合当前真机验收阶段，能稳定暴露三类道具效果。
- `6` 个上限可以增加争夺机会，但不会让拾取检测和绘制失控。

## 原版拾取规则

原版在实体移动后调用 `collectPowerUps(entity, now)`。

拾取判断：

```text
distance(entity, powerUp) <= entity.radius + powerUp.radius
```

命中后：

- 道具从场上移除。
- 如果 `teamPowerUps = false`，只给拾取单位应用效果。
- 如果 `teamPowerUps = true`，给同阵营所有存活单位应用同一个效果。

原版事件：

| 条件 | 事件文案 |
| --- | --- |
| 单体拾取 | `<阵营> 获得<道具名>` |
| 团队共享 | `<阵营> 全队获得<道具名>` |

## 原版效果规则

### 加速

原版行为：

- `rageUntil = max(rageUntil, now + 6000)`
- `vx *= 1.22`
- `vy *= 1.22`
- 移动限速时，如果仍在 `rageUntil` 内，最大速度提升到 `MAX_SPEED * 1.25`

迁移建议：

- 新版不要只乘当前速度，否则不同帧率和不同状态下体感会漂。
- 建议给实体增加 `speedBoostUntilMs`，移动系统根据该时间统一套速度上限。
- 可保留一次性速度推动，但必须受 `MAX_SPEED` 上限约束。

### 护盾

原版行为：

- `shield = max(shield, 1)`
- 在碰撞转化时抵挡一次被转化。

迁移建议：

- 在新版碰撞系统里明确处理护盾：
  - 攻击方克制防守方时，如果防守方 `shield > 0`，消耗一层护盾，不转阵。
  - 只有无护盾时才执行原有转阵逻辑。
- Canvas 上需要给带护盾单位轻量描边或外圈，避免玩家看不懂为什么没被转化。

### 分裂

原版行为：

- 在拾取单位附近生成一个同阵营单位。
- 出生坐标为拾取单位 `x/y ± 18`。
- 出生半径倍率为 `0.86`。

迁移建议：

- 使用新版实体生成入口，不直接在系统里拼实体对象。
- 需要接入当前人数上限，避免团队道具下“一次拾取全队分裂”导致实体数暴涨。
- 如果总人数已到上限，分裂道具可以只播放提示，不继续生成。

## 团队道具的特殊风险

`teamPowerUps` 的原版语义是“全队共享效果”，不是“只给队友一个小加成”。这会带来几个明显风险：

- 团队加速会让整队速度同时提升，追捕节奏会明显变快。
- 团队护盾会让整队短时间抗转化，可能改变据点争夺节奏。
- 团队分裂会对同阵营每个存活单位执行一次分裂，人数可能瞬间翻倍。

新版必须加上性能和人数保护：

- 场上道具数量限制为最多 `6`。
- 道具拾取检查最多是 `实体数 * 6`，在当前规模内可接受。
- 分裂必须受全局实体上限约束。
- 团队分裂建议加每次拾取生成上限，避免一帧大量对象创建。
- 道具效果、拾取提示和绘制都走 Canvas / 引擎状态，不做每帧 ArkUI 状态刷新。

## 原版更新顺序

原版 `GameEngine.update` 的相关顺序：

```text
updateBlackHole
updateTraitors
updateThanos
updateRevives
updatePowerUps
updateEntities
resolveCollisions
updateZones
updateLastStand
checkFinish
```

其中 `collectPowerUps` 在 `updateEntities` 内部，实体完成移动、边界和障碍处理后执行。

这意味着：

- 道具刷新/过期发生在本帧移动前。
- 实体移动后立刻能拾取道具。
- 道具效果会影响本帧之后的碰撞、占点和后续帧移动。
- 原版黑洞力在 `updateEntities` 内也会参与移动，所以黑洞影响和拾取在同一轮实体更新里发生。

新版建议顺序：

```text
powerUpSystem.updateSpawnAndExpire
movement
powerUpSystem.collect
blackHole
collision
zone
lastStand
result
```

当前新版黑洞系统已经独立在移动后执行，不能完全一比一复刻原版内部顺序。第一版应优先保证玩家感知和性能稳定：移动后拾取，道具效果再影响本帧后续结算和下一帧移动。

新版视觉建议：

- 不直接照搬旧版高饱和颜色，改用当前战术桌面主题下的清晰小物件。
- 加速用冷蓝电光，护盾用金色护环，分裂用紫色双影。

- 拾取时只做短促脉冲，不做大面积遮罩。

## 原版 UI 依赖

旧版 `精彩事件` 面板中有两个按钮：

```text
能量道具：加速/护盾/分裂
团队道具：全队共享效果
```

旧版点击规则：

- 点击关闭 `能量道具` 时，自动关闭 `团队道具`。
- 点击打开 `团队道具` 时，自动打开 `能量道具`。

新版机制卡应保留这个关系：

| 机制 | 状态 | 默认 | 是否可关闭 | 依赖 |
| --- | --- | --- | --- | --- |
| 中心据点 | 核心 | 开 | 不可关闭 | 无 |
| 黑洞 | 增强 | 开 | 可关闭 | 无 |
| 绝地求生 | 增强 | 开 | 可关闭 | 无 |
| 能量道具 | 增强 | 开 | 可关闭 | 无 |
| 团队道具 | 增强 | 开 | 可关闭 | 依赖能量道具 |

交互规则：

- `能量道具` 关闭后，`团队道具` 显示为关。
- `团队道具` 打开时，如果 `能量道具` 处于关闭状态，则先自动打开 `能量道具`。
- 战斗开始后不允许改机制，避免运行时状态不一致。

## 性能日志

迁移后建议在 `RpsBattlePerf` 追加或复用这些字段：

```text
powerUpMs
powerUpsActive
powerUpsPicked/frame
powerUpEffects/frame
teamPowerUpEffects/frame
```

验收目标：

- 大局 `36` 实体下，道具系统平均耗时应接近 `0.05ms` 量级。
- 不因为团队分裂造成实体数无限增长。
- 关闭 `能量道具` 后，`powerUpMs` 接近 `0`，场上不生成道具。
- 开启 `团队道具` 后，拾取事件明显显示“全队获得”，但 FPS 不应出现稳定性下降。

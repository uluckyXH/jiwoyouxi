# 石头剪刀布大作战视觉主题升级设计

日期：2026-07-21
状态：第一版已实现，待真机视觉验收
适用范围：新版 `entry/src/main/ets/games/rpsBattle/` 的配置页、战斗页、HUD、暂停与结算弹窗

## 结论

当前新版乱斗的视觉基调是大面积米黄、浅橙和深棕。它和鸡窝 App 的温暖感一致，但在石头剪刀布大作战里覆盖面积过大，导致页面偏黄、层级发糊、战斗感不足。

推荐把新版乱斗调整为 **清爽战术桌面** 主题：

- 页面底色从米黄改为浅冷灰蓝，减少整屏发黄。
- 主操作从深棕改为深青绿，强化“开始战斗”的行动感。
- 棕色只保留在小徽章、标题局部和品牌点缀上，维持鸡窝 App 统一感。
- 模式和机制使用不同强调色，给“经典乱斗、团队占点、黑洞”等玩法建立清晰身份。
- Canvas 战场使用浅色棋盘底和低透明网格，实体、据点、黑洞成为视觉焦点。

本轮只做主题和视觉层级升级，不改变玩法规则、人数配置、机制开关、性能循环和旧版 `rps`。

## 实施记录

2026-07-21 已完成第一版主题落地：

- `RpsBattleTheme.ets` 已从米黄/棕色主题切到浅灰蓝背景、白色半透明卡片、深青绿主操作和青灰描边。
- 配置页的玩法、机制、阵营主题、支持阵营、人数、时长选项已替换旧浅黄底色。
- 玩法方案选中态改为浅底 + 模式强调色描边：经典乱斗青绿、团队占点蓝绿、小心叛徒危险色、众生平等紫色。
- 顶部栏、底部开始区、暂停弹窗、结算弹窗、倒计时遮罩和 HUD 已统一为新主题。
- Canvas 战场底色、网格、外框、实体阴影、黑洞吸引圈已适配浅色战术桌面。
- 默认 RPS 阵营色改为石头灰蓝、剪刀青绿、布琥珀；其他主题保留原有阵营识别色。

本次没有改动玩法规则、黑洞开关、团队占点规则、倒计时状态机、人数规则和性能主循环。

## 当前问题

### 1. 黄色面积过大

现有 `RpsBattleTheme.ets` 中主要颜色为：

| Token | 当前值 | 问题 |
| --- | --- | --- |
| `pageBg` | `#f6ead6` | 整页偏黄，和空当接龙、纸牌类游戏气质接近 |
| `topBarBg` | `#fff0d5` | 顶部栏继续加深暖黄面积 |
| `panelBg` | `rgba(255,248,232,0.92)` | 卡片和背景色相接近，层级靠深浅区分 |
| `primaryButtonBg` | `#563723` | 深棕按钮偏传统，不够像战斗启动 |
| `selectedBg` | `#fff4d9` | 选中态仍然是黄底，和卡片底缺少区分 |

### 2. 玩法辨识度不足

经典乱斗、团队占点、黑洞机制本质是不同强度和不同气质的玩法，但目前主要共享棕黄色选择态。玩家能读懂文字，但第一眼不容易从颜色和结构判断“这一局是什么类型”。

### 3. 配置页和战斗页气质不统一

配置页像暖色卡片表单，战斗页是 Canvas 追捕模拟。两者之间缺少共同的战术桌面感，导致“设置”和“开战”之间的情绪断层比较明显。

## 视觉定位

### 推荐定位

```text
清爽战术桌面

浅冷底色 + 青绿主操作 + 少量鸡窝暖色 + 阵营高饱和点缀
```

关键词：

- 清爽：减少大面积黄底，手机上看久不腻。
- 战术：卡片像战术配置板，玩法、机制、阵营有明确分区。
- 桌面：不做纯暗黑电竞，不牺牲当前 App 的亲和感。
- 阵营：让石头、剪刀、布和模式机制靠颜色形成记忆点。

### 不采用的方向

| 方向 | 不采用原因 |
| --- | --- |
| 纯暗黑电竞 | 和鸡窝游戏整体轻松风格差异过大，小屏文字对比和弹窗会变重 |
| 继续米黄纸张 | 现在的问题正是暖黄面积过大 |
| 大面积紫蓝渐变 | 容易变成通用科技感，不像当前 App，也不利于机制和阵营分色 |
| 原版完全复刻 | 旧版可以作为玩法参考，但新版需要更清楚的配置层级和移动端阅读体验 |

阵营色继续跟随主题配置，但默认 RPS 建议更稳一点：

| 阵营 | 建议色 | 用途 |
| --- | --- | --- |
| 石头 | `#64748b` | 单位、支持阵营选中边、战报图标 |
| 剪刀 | `#0ea5a4` | 单位、选中态、战报图标 |
| 布 | `#f59e0b` | 单位、选中态、战报图标 |

这三个色值的饱和度不同，能在浅色战场上拉开差异，也不会全部挤在黄棕色系里。

## 组件设计规则

### 顶部栏

- 未开局配置页：标题居中，左侧“大厅”保留，右侧占位保持对称。
- 战斗中：标题允许更短，左侧大厅和右侧暂停都可见。
- 顶部栏背景使用 `rgba(255,255,255,0.76)`，不再使用暖黄。
- 按钮底色使用 `rgba(15,118,110,0.10)`，文字和图标使用深青绿。

### 配置卡

- 卡片背景：白色半透明。
- 卡片圆角沿用项目规范，不超过 `8vp`。
- 卡片之间使用真实间距，不靠阴影堆层级。
- 每张卡只解决一个配置问题，避免卡片套卡片。
- 编号徽章使用小面积品牌暖棕或模式色，不再使用大块黄底。

### 选中态

选中态统一为：

```text
白底 + 青绿描边 + 轻色填充 + 小色条 / 角标
```

不同玩法可以换小色条颜色：

| 玩法 | 强调色 |
| --- | --- |
| 经典乱斗 | `#0f766e` |
| 团队占点 | `#256f9c` |
| 小心叛徒 | `#b4533f` |
| 众生平等 | `#7c3aed` |

强调色只用于选中边、角标、局部图标，不铺满整张卡。

### 底部开始区

- 固定底部动作区保留，但背景从米黄改为半透明白。
- 摘要行使用深青灰文字，关键信息使用模式强调色。
- “开始乱斗”按钮用深青绿，文字白色，按钮高度保持稳定。

### 暂停与确认弹窗

- 弹窗背景白色半透明，标题深青黑。
- “继续对局”和“重新开始”使用主色。
- “确认回大厅”使用危险色。
- 弹窗不使用纯黄底，避免和配置页混成同一层。

### Canvas 战场

- 背景从暖黄改为浅白绿。
- 网格线更淡，只辅助方向感。
- 据点使用蓝绿环形和进度描边。
- 黑洞使用深紫黑核心、低透明吸引圈，移动端继续遵守现有半径上限。
- 实体颜色优先级最高，不能被网格、据点和黑洞抢掉。

## 验收标准

视觉验收：

- 首页进入新版乱斗后，首屏不再是大面积米黄。
- 配置页一眼能看清当前选中的玩法、机制和开始按钮。
- `团队占点` 和 `经典乱斗` 的视觉身份有明显差异。
- 手机小屏上标题、按钮、模式卡文字不截断、不重叠。
- 暂停弹窗和返回大厅确认弹窗符合新版主题。

玩法验收：

- 经典乱斗、团队占点、黑洞开关、人数调整、倒计时、暂停、重新开始、重新选择行为不变。
- 旧版 `rps` 入口继续可见。
- 未实现机制仍然不能点击开启。

性能验收：

- 主题改造不引入新的每帧 ArkUI 状态更新。
- `RpsBattlePerf` 日志继续正常输出。
- 大局 36 实体下，视觉调整不明显抬高 `draw` 和 `update` 耗时。

## 注意事项

- 不要把整个页面改成单一青绿色系。青绿只做主操作和选中态，阵营和机制要有自己的颜色。
- 不要新增 WebView、HTML、CSS 动画或外部图片依赖。
- 不要为了视觉效果给 Canvas 增加大量阴影、渐变和复杂路径。
- 不要把主题色硬编码散落到组件中，优先走 `RpsBattleTheme.ets`。
- 如果某个组件需要模式强调色，优先从模式配置或明确 helper 中读取，不在多个文件重复判断模式名。

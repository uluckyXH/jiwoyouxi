# 阵营主题与阵容选择适配设计

日期：2026-07-09
状态：设计稿
适用范围：新版 `entry/src/main/ets/games/rpsBattle/`

## 背景

新版 `rpsBattle` 已完成基础乱斗、倒计时开战、重开、音效和性能优化。下一步需要补齐游戏内容，让开局前的选择更有“阵营感”和策略感。

旧版 `games/rps` 已经有阵营主题体系，代码来源主要是：

- `entry/src/main/ets/model/ThemeConfig.ets`
- `entry/src/main/ets/games/rps/RpsPage.ets` 的 `ThemePanel()`、`LineupPanel()`

新版不直接照搬旧版所有设置项。当前阶段只做阵营主题和阵容选择，不做自定义改名，不加入旧版复杂事件开关。

## 原版阵营主题

旧版 `ThemeConfig.ets` 提供 4 套主题，每套主题包含 `key`、`name`、`mark`、`subtitle` 和 3 个阵营信息。

| key | 名称 | 阵营 |
| --- | --- | --- |
| `rps` | RPS | 石头、剪刀、布 |
| `colors` | 颜色球 | 红球、绿球、蓝球 |
| `countries` | 国家队 | 中国队、美国队、巴西队 |
| `brands` | 品牌战 | Nova、Pulse、Orbit |

每个阵营有固定的名称、符号、主色和深色。旧版还允许非 `rps` 主题自定义名称，但新版这一步不做改名，避免选择界面变重，也避免后续战报、音效和素材命名出现不稳定文本。

## 新版适配目标

1. 保留三阵营循环克制的底层关系，不因为主题变化而改变规则。
2. 主题只影响展示：阵营名称、图标、颜色、HUD、棋子、结算弹窗。
3. 开局前只提供清晰的阵营和阵容选择，不提供自定义名称输入。
4. 选择流程要短：玩家能在一屏内完成“主题、支持阵营、阵容规模、开始乱斗”。
5. 代码继续保持 `rpsBattle` 包内自洽，不把旧版页面状态直接搬进新版。

## 选择界面结构

建议将当前 `RpsBattleSetupPanel` 调整为 4 个紧凑区域。

### 1. 主题选择

显示为 2x2 网格，每个主题是一个可点击块：

- 第一行：主题符号，例如 `🪨✂️✋🏻`、`🔴🟢🔵`
- 第二行：主题名称，例如 `RPS`、`颜色球`
- 选中态：深色底 + 白色文字
- 未选中态：浅色底 + 边框

点击主题后：

- 更新当前主题。
- 支持阵营自动保持同一阵营位序，例如原来支持第 2 位，切到国家队后仍支持第 2 位的美国队。
- HUD 和棋盘预览立即更新颜色和图标。
- 不出现名称输入框。

### 2. 支持阵营

使用当前三阵营卡片样式，但内容来自所选主题：

- RPS：石头、剪刀、布
- 颜色球：红球、绿球、蓝球
- 国家队：中国队、美国队、巴西队
- 品牌战：Nova、Pulse、Orbit

交互规则：

- 点击卡片即选中支持阵营。
- 选中卡片使用主题阵营主色描边。
- 卡片内展示图标、名称和克制说明。

克制说明建议固定用位序表达，不把文案写死成“石头克剪刀”：

- 第 1 位克第 2 位
- 第 2 位克第 3 位
- 第 3 位克第 1 位

展示时可转成主题名，例如 `中国队克美国队`。如果空间不够，只显示 `克 美国队`。

### 3. 阵容规模

保留当前 `小局 / 标准 / 大局` 三档，但展示更明确：

| 档位 | 数量 | 用途 |
| --- | --- | --- |
| 小局 | 5 / 5 / 5 | 快速看结果 |
| 标准 | 8 / 8 / 8 | 默认推荐 |
| 大局 | 12 / 12 / 12 | 更热闹，适合观察翻盘 |

当前阶段不开放手动输入每方数量。原因：

- 大作战当前重点是流畅体验和主题表达。
- 自定义数量会带来性能、布局和结算平衡问题。
- 后续如果需要，再单独做“高级阵容”。

### 4. 开始乱斗

按钮仍保持单一主操作。

点击后：

1. 进入战斗界面。
2. 显示 3、2、1 开战倒计时。
3. 倒计时结束才启动移动和正式计时。

## 战斗界面适配

主题需要贯穿以下位置：

- HUD 三阵营数量卡：图标、名称、颜色。
- HUD 支持阵营：显示当前主题名称。
- Canvas 棋子：使用主题图标和主色。
- 转阵脉冲：使用转换后阵营颜色。
- 结算弹窗：显示获胜阵营图标和名称。
- 音效不随主题变化，先复用现有音效。

当前 Canvas 棋子使用 `faction.symbol` 和 `faction.color`，适合直接接入主题数据。

## 数据设计

建议在 `rpsBattle` 包内新增独立配置文件：

`entry/src/main/ets/games/rpsBattle/RpsBattleThemeConfig.ets`

原因：

- 新版包保持独立，避免直接依赖旧版 `model/ThemeConfig.ets` 的 `NativeGameOptions` 和自定义名称逻辑。
- 可以只保留新版需要的字段。
- 后续加入素材图、阵营皮肤、专属音效时不影响旧版。

建议类型：

```ts
export type RpsBattleThemeKey = 'rps' | 'colors' | 'countries' | 'brands';

export interface RpsBattleThemePreset {
  key: RpsBattleThemeKey;
  name: string;
  mark: string;
  subtitle: string;
  factions: RpsBattleFactionInfo[];
}
```

规则层仍使用固定的三类逻辑 id：

- `rock`
- `scissors`
- `paper`

主题层只把这三个逻辑 id 显示成不同阵营。这样最稳，不需要改碰撞、克制、转阵和结算规则。

## 页面状态建议

`RpsBattlePage` 增加：

```ts
@State selectedThemeKey: RpsBattleThemeKey = 'rps';
```

`RpsBattleSetupPanel` 增加：

```ts
@Prop selectedThemeKey: RpsBattleThemeKey = 'rps';
selectTheme: (themeKey: RpsBattleThemeKey) => void = () => {};
```

引擎 `RpsBattleEngine` 不需要知道主题。页面创建引擎时仍传 `supportFaction`、`scale`、`durationSec` 等规则参数。渲染器和 HUD 接收当前主题下的 `factionInfo`。

## 交互细节

### 主题切换

- 切换主题不重置规模。
- 切换主题不重置支持阵营位序。
- 切换主题后，开始按钮文案不变。
- 切换主题时不播放音效，避免选择界面太吵。

### 支持阵营切换

- 点击即选中。
- 可以播放轻量点击音效，后续再接。
- 支持阵营只影响结算反馈和玩家代入，不改变 AI 行为。

### 阵容规模切换

- 点击即选中。
- 默认 `标准`。
- 选择 `大局` 时不弹性能提示，当前性能已经优化到可运行；后续如果加入更多元素再考虑提示。

### 重开

战斗中点击 `重开`：

- 使用当前主题、支持阵营和规模重新生成一局。
- 重新进入 3、2、1 倒计时。
- 不返回选择界面。

## 实施步骤

1. 新增 `RpsBattleThemeConfig.ets`，复制并收敛旧版 4 套主题。
2. 将 `RpsBattleRules.rpsBattleFactionInfo()` 改为可接收当前主题，或新增主题查询函数供 UI 和渲染层使用。
3. `RpsBattleSetupPanel` 增加主题选择网格。
4. `RpsBattleHud` 使用当前主题显示三阵营。
5. `RpsBattleRenderer` 使用当前主题绘制棋子。
6. `RpsBattleResultOverlay` 使用当前主题展示获胜方和统计。
7. 保持 `RpsBattleEngine` 规则层不依赖主题。

## 验收标准

- 选择界面可切换 4 套主题。
- 每套主题下，三阵营图标、名称、颜色在选择页、HUD、棋子、结算弹窗一致。
- 不出现任何自定义名称输入框。
- 切换主题后支持阵营位序保持稳定。
- 点击开始乱斗后仍进入倒计时，再进入正式战斗。
- 重开沿用当前选择，不回到默认主题。
- `./scripts/build-app.sh --stacktrace` 通过。

## 暂不做

- 自定义阵营名称。
- 自定义每方人数。
- 主题专属音效。
- 主题专属素材图。
- 把旧版所有预设、事件开关搬入新版。

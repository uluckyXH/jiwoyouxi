# 石头剪刀布大作战主玩法设计

日期：2026-07-08
状态：根据现有代码梳理

## 一、游戏定位

石头剪刀布大作战是一个三阵营自动乱斗游戏。玩家开局配置三个阵营的初始人数和特殊规则，进入战场后，单位会自动移动、追捕、躲避和碰撞。阵营之间遵循石头剪刀布循环克制，碰撞后败方单位会被胜方转化。

现有游戏更像一个“自动模拟观战器”，玩家的主要乐趣来自：

- 开局押注或选择支持阵营。
- 配置不同事件组合，看局势如何变化。
- 观察强弱关系、追捕链条和突然翻盘。
- 赛后查看战报，理解本局关键节点。

新版重构时建议强化为“可读性更强的阵营乱斗”。核心不是让玩家精准操作每个单位，而是让玩家清楚看懂谁在追谁、谁被克制、为什么翻盘。

## 二、基础规则

### 2.1 三阵营

默认阵营：

| 类型 | 阵营 | 克制 | 被克制 |
| --- | --- | --- | --- |
| `0` | 石头 | 剪刀 | 布 |
| `1` | 剪刀 | 布 | 石头 |
| `2` | 布 | 石头 | 剪刀 |

代码里的克制函数是：

```text
石头克剪刀，剪刀克布，布克石头
```

底层以 `type` 表示阵营。对任意单位：

- 猎物是 `(type + 1) % 3`。
- 天敌是 `(type + 2) % 3`。

这意味着单位会主动追击自己能克制的阵营，同时在近距离躲避能克制自己的阵营。

### 2.2 单位行为

每个单位是一个圆形实体，拥有：

- `id`：唯一编号。
- `type`：所属阵营。
- `x/y`：位置。
- `vx/vy`：速度。
- `radius`：半径。
- `speed`：基础速度。
- `shield`：护盾次数。
- `alive`：是否存活。
- `rageUntil`：加速或暴走状态结束时间。

每帧更新时，单位会依次处理：

1. 寻找最近猎物并向其加速。
2. 寻找最近天敌，距离越近越主动躲避。
3. 如果悬赏开启，非头名阵营会额外追击头名。
4. 受到黑洞、上帝之手、道具等外力影响。
5. 限制最大速度。
6. 移动并处理边界、障碍、道具拾取。
7. 与其他单位碰撞并结算转化。

## 三、碰撞转化

两个不同阵营单位发生碰撞时：

1. 先做物理分离和速度交换，避免重叠。
2. 如果触发“我要打十个”，可能改为直接击杀优势方。
3. 否则按循环克制关系判断胜负。
4. 败方单位的 `type` 变成胜方阵营。
5. 败方获得短暂 `rageUntil`，速度略放大，让画面有冲击反馈。
6. 如果败方原本是悬赏头名，胜方触发悬赏奖励。
7. 战报记录这次转化和转化贡献者。

护盾会拦截一次转化。单位有护盾时，碰撞失败不会改阵营，只消耗一层护盾。

## 四、玩家流程

现有页面状态：

| 状态 | 含义 |
| --- | --- |
| `Setup` | 配置界面 |
| `Countdown` | 开局倒计时 |
| `Playing` | 对局进行中 |
| `Paused` | 暂停 |
| `Result` | 结算 |

标准流程：

1. 玩家在配置页选择预设、主题、人数、时长、机制、事件、赛制。
2. 点击“开始乱斗”。
3. 如果开启下注，倒计时页可选择支持阵营。
4. 倒计时 `3、2、1、开始` 后进入战场。
5. 游戏自动模拟，玩家可暂停、回首页，开启上帝之手时可触摸干预。
6. 触发胜负条件后进入结算，1 秒后展示结果卡。
7. 玩家可查看战报、再来一局、进入下一局或回首页。

## 五、默认配置

默认选项来自 `defaultOptions()`：

| 项目 | 默认值 |
| --- | --- |
| 阵营主题 | `rps` |
| 初始人数 | `7/7/7` |
| 时长 | `60s` |
| 死斗 | 关闭 |
| 多局赛制 | 关闭 |
| 上帝之手 | 关闭 |
| 障碍、缩圈、悬赏、黑洞、叛徒等 | 全部关闭 |
| 下注 | 关闭 |
| 下注本金 | `100` |
| 通知 | 开启 |

人数范围在页面操作和引擎里都会限制到 `3..60`。在宽屏设备预设会使用 `30/30/30`，手机宽度下预设使用 `7/7/7`。

## 六、玩家可直接影响的内容

现有玩法中，玩家不是实时操控某个阵营，直接操作主要有三类：

### 6.1 开局配置

- 三方初始人数。
- 是否同步三方人数。
- 对局时长。
- 游戏机制和事件开关。
- 死斗、多局、下注。
- 阵营主题和自定义名称。

### 6.2 下注选择

开启下注后，玩家在倒计时前后都可以选择支持阵营。胜利后按赔率获得收益，失败扣除下注额。

### 6.3 上帝之手

开启后，玩家触摸战场会产生力场：

- `推开`：把触摸点附近单位向外推。
- `吸引`：把触摸点附近单位向内吸。

当前范围约为 `130px`，力度由距离衰减。

## 七、重构目标

新版应保留：

- 三阵营循环克制。
- 自动追捕和躲避。
- 碰撞转化。
- 清场、限时和据点胜利。
- 关键翻盘事件。
- 战报和走势总结。

新版应改进：

- 减少配置页开关堆叠，优先做几个强体验模式。
- 战场表达要更清楚，让玩家能看懂追击链。
- 单位和事件需要更主题化，不再只是圆点和基础图形。
- 结果页要突出“为什么赢”，而不是只显示人数。
- 规则层、表现层、页面层拆开，为后续扩展留空间。

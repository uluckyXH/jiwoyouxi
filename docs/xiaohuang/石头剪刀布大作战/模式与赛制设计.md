# 石头剪刀布大作战模式与赛制设计

日期：2026-07-08
状态：根据现有代码梳理

## 一、模式结构

现有游戏的“模式”由几类配置组合而成：

- 预设：一键套用多个机制。
- 基础赛制：限时或死斗。
- 多局赛制：三局两胜或五局三胜。
- 下注积分：玩家选择支持阵营。
- 主题身份：改变阵营外观和名称。

这些配置都在 `RpsPage.ets` 中维护状态，最终通过 `currentOptions()` 传给 `NativeGameEngine.start()`。

## 二、预设

预设来自 `Presets.ets`。

| key | 名称 | 配置行为 | 设计含义 |
| --- | --- | --- | --- |
| `classic` | 经典大乱斗 | 开启死斗 | 三派同场，直到一方清场 |
| `zones` | 团队占点 | 据点、黑洞、绝地求生、道具、团队道具 | 围绕中心据点争夺，事件制造翻盘 |
| `traitor` | 小心叛徒 | 缩圈、黑洞、叛徒、死斗 | 空间越来越小，叛徒改写阵营 |
| `equality` | 众生平等 | 灭霸响指、障碍、死斗 | 随机减半人数，障碍制造混乱 |

预设只是一组开关，不是独立规则引擎。玩家手动切换任何选项后，页面会把 `selectedPreset` 改为 `custom`。

## 三、限时模式

默认模式是限时模式：

- 默认时长 `60s`。
- 页面按钮每次把时长增加 `15s`。
- 达到 `120s` 后再点击回到 `30s`。
- 到时后，如果存在唯一领先阵营，该阵营获胜。
- 如果到时人数并列，`leadingType()` 返回 `-1`，不会立刻产生唯一赢家。

限时模式适合重构新版首屏，因为节奏可控，结果页能稳定出现。

## 四、死斗模式

死斗模式开启后：

- `timeLeftSec` 显示为 `-1`。
- HUD 状态显示“死斗中”。
- 不按时长结束。
- 只有清场、据点达标或特殊机制导致胜利时才结束。

死斗适合“经典大乱斗”“小心叛徒”“众生平等”这类看最终霸屏的模式。

## 五、多局赛制

多局赛制由三个状态控制：

- `tournament`：是否开启。
- `bestOf`：`3` 或 `5`。
- `winnerPenalty`：上一局胜方下一局减员数，取值按钮循环 `0/5/10/15`。

结算时：

1. 如果本局有赢家，给赢家阵营胜场加 `1`。
2. 给赢家阵营累计 `winnerPenalty`。
3. 下一局初始人数使用 `currentRoundCounts()`：

```text
下一局人数 = max(3, 配置人数 - 累计惩罚)
```

4. 当某阵营胜场达到 `ceil(bestOf / 2)`，整场结束。

多局赛制的设计含义是“强者削弱”，让连续获胜的一方下一局更难，增加看点。

## 六、下注积分

下注模式相关状态：

| 字段 | 含义 | 默认值 |
| --- | --- | --- |
| `bank` | 玩家积分余额 | `1000` |
| `stake` | 当前下注额 | `100` |
| `selectedPick` | 支持阵营 | `0` |
| `oddsRock/Scissors/Paper` | 三方赔率 | 根据人数计算 |

下注只影响本地积分，不影响战斗模拟。

### 6.1 赔率

赔率按三方人数计算，核心公式：

```text
strength = count[type] + preyCount * 0.3 - predatorCount * 0.18
odds = total / strength * 0.72
```

再限制到：

```text
1.18 <= odds <= 4.8
```

含义：

- 自身人数越多，赔率越低。
- 猎物多，对自身有利，赔率降低。
- 天敌多，对自身不利，赔率升高。

### 6.2 结算

如果猜中赢家：

```text
收益 = round(stake * (odds - 1))
bank += 收益
```

如果猜错：

```text
bank -= stake
```

如果无人获胜，不结算下注。

重构建议：下注可以保留为“观战选择”或“应援积分”，避免做成强赌博感。文案可以偏轻，比如“支持”“应援”“鸡蛋票”。

## 七、主题身份

当前支持四套主题：

| key | 名称 | 阵营身份 |
| --- | --- | --- |
| `rps` | RPS | 石头、剪刀、布 |
| `colors` | 颜色球 | 红球、绿球、蓝球 |
| `countries` | 国家队 | 中国队、美国队、巴西队 |
| `brands` | 品牌战 | Nova、Pulse、Orbit |

主题只改变阵营标签、图标、颜色，不改变克制关系。非 `rps` 主题可以自定义三个阵营名称。

重构建议：新版可以保留主题系统，但首版重点应放在一个强主题上，不要把主题选择作为主要玩法。

## 八、模式重构建议

新版不建议把所有开关直接平铺给玩家。推荐先做三个明确入口：

| 新版入口 | 建议继承内容 | 目的 |
| --- | --- | --- |
| 经典乱斗 | 三方均衡、限时或死斗、无复杂事件 | 让玩家快速理解核心规则 |
| 翻盘事件 | 黑洞、叛徒、绝地求生、道具 | 展示戏剧性和随机性 |
| 据点争夺 | 中心据点、团队道具、悬赏 | 形成明确战术目标 |

高级配置可以作为调试或“自定义房间”，不作为首屏主要体验。

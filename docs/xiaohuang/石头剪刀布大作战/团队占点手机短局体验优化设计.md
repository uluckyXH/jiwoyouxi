# 石头剪刀布大作战团队占点手机短局体验优化设计

日期：2026-07-24
状态：P1/P2 轻量版已实现，待真机验收
适用范围：新版 `entry/src/main/ets/games/rpsBattle/` 的 `团队占点` 模式

## 实施记录

2026-07-24 已完成第一版轻量实现：

- `RpsBattleEngine.ets` 中团队占点开局前 `4s` 提高部分单位的中心据点意图，让手机短局一开始更像“抢点”。
- 开局强化仍保留天敌逃离优先级，单位附近有克制自己的阵营时会降低占点意图，不改石头剪刀布基础追逃。
- `RpsBattleRenderer.ets` 中心据点增加开局目标脉冲。
- 据点 Canvas 增加三方压力点和压力数字，帮助观察谁在圈内争夺。
- 僵持状态使用中性色边界，不误导玩家以为正在稳定涨分。
- 新增 HarmonyOS Hypium 测试文件 `RpsBattleZoneSystem.test.ets`，覆盖圈内压力统计、唯一领先涨分和僵持不涨分。
- 本轮未改清场结算、据点分数模型、黑洞出现时间、道具生成策略和多据点。

## 结论

`团队占点` 可以做一轮类似 `小心叛徒` 的手机短局体验优化，但优化重点不同。

`小心叛徒` 的问题是核心事件触发太晚，手机端可能还没看到叛徒就清场。`团队占点` 当前已经不按清场立即结束，问题更偏向：

- 玩家是否一开局就能感受到“抢据点”是主线。
- 单位是否稳定围绕中心据点拉扯，而不是又滑回普通吞噬。
- 黑洞、绝地求生、能量道具、团队道具是否服务于占点，而不是抢走占点焦点。
- 手机竖屏空间小，中心据点、道具和黑洞如果节奏不整理，会显得乱。

所以这轮不是加更多机制，而是把已有机制重新排节奏，让玩家更早、更清楚地看到团队占点的目标。

## 当前实现基线

当前 `团队占点` 已经不是占位模式，已有以下能力：

| 能力 | 当前状态 |
| --- | --- |
| 中心据点 | 核心机制，不可关闭 |
| 黑洞 | 默认开启，可关闭，移动端有动态上限 |
| 绝地求生 | 默认开启，可关闭，弱势方分裂和援军 |
| 能量道具 | 默认开启，可关闭，包含加速、护盾、分裂 |
| 团队道具 | 默认开启，可关闭，拾取后全队共享效果 |
| 清场结算 | 团队占点不再因只剩一方立即结束 |
| 胜利方式 | 据点到 `100` 或时间到领先 |
| 性能日志 | 已有 `zoneMs`、`blackHoleMs`、`lastStandMs`、`powerUpMs` 等分段 |

当前代码上的关键数值：

| 项 | 当前值 |
| --- | --- |
| 据点满分 | `100` |
| 据点得分 | 圈内唯一领先阵营人数 `* 2.8 * dt` |
| 据点半径 | 短边 `15%`，限制在 `44..62` |
| 黑洞首次出现 | `9s..16s` |
| 道具首次出现 | `3s` |
| 道具间隔 | `4s` 加少量随机 |
| 最大道具数 | `6` |
| 援军触发 | 阵营剩余 `<= 3` 且人数下降或清零 |
| 援军延迟 | `2.2s` |
| 援军冷却 | `5s` |

这些能力已经足够做可玩性调优，下一步不应该先加新模式，而应该整理体验顺序。

## 手机端问题判断

### 1. 开局目标不够强

目前据点在中心，但战斗开始后单位仍会受基础追捕、逃离天敌、黑洞、道具影响。虽然已经有占点 AI，但玩家未必能在前几秒明确感受到“大家是在抢据点”。

需要让开局前 `3s..5s` 成为“抢中心”的强信号：

- 单位更积极向中心聚集。
- 据点视觉更醒目。
- HUD 更强调当前据点归属和压力。
- 第一个道具最好不要把单位从据点附近拉得太远。

### 2. 机制都开时信息过多

团队占点默认包含五个机制。手机竖屏上同时出现中心据点、黑洞、复活印记、道具、分裂、护盾、速度效果，容易变成“哪里都在发生事情”，反而弱化占点。

需要给机制排序：

```text
据点永远是主线
援军用于续航
道具用于局部翻盘
黑洞用于扰乱站位
吞噬只是基础冲突
```

### 3. 单一据点分数解释仍偏弱

当前据点只有一个 `owner + score`。这接近原版，但手机端观察时可能会出现：

- 后手抢到 owner 后，像是继承了之前的共同进度。
- 玩家不知道其他阵营在据点上贡献了多少。
- 时间结束时如果不是 `100` 分结算，领先逻辑不够直观。

这不一定马上改，但需要作为下一阶段备选。

### 4. 黑洞和道具需要服务占点

黑洞如果太靠中心，会打断占点；太边缘，又像无关事件。道具如果随机刷太边缘，会把单位吸离主战场。

下一轮可以先不改机制本身，只改生成策略：

- 道具优先刷在据点外围，不刷在极远角落。
- 黑洞避开据点核心，但可以出现在据点外侧，制造绕路和拉扯。
- 复活点继续避开黑洞。

## 设计目标

1. 手机标准局开局 `5s` 内就能看出各阵营在抢中心据点。
2. 据点进度增长更可感知，玩家能理解谁在占、为什么涨分。
3. 弱势方仍能回场，但不会无限拖局。
4. 黑洞和道具制造变化，但不喧宾夺主。
5. 不影响 `经典乱斗`、`小心叛徒`、`众生平等`。
6. 继续使用 HarmonyOS 原生 ArkTS / ArkUI / Canvas，不引入 WebView、HTML、Node 运行依赖或跨端壳。
7. 按 `docs/鸿蒙原生开发规范.md` 拆分规则系统、渲染、测试和报告。

## 推荐优化方案

### P1：开局抢点节奏

先做低风险的开局节奏调优。

| 项 | 建议 |
| --- | --- |
| 开局占点强化窗口 | 前 `4s` |
| 争夺者占点意图 | 前 `4s` 小幅提高 |
| 守卫者占点意图 | 本方占领后更稳定留在圈边 |
| 天敌逃离优先级 | 保持高于占点，避免单位无脑送 |
| 据点视觉 | 开局轻脉冲，强调目标 |

实现原则：

- 不加 ArkUI 状态刷新，只在引擎移动向量和 Canvas 绘制层做轻量调整。
- 角色分配继续使用 `entity.id`，不每帧随机。
- 不改变基础克制规则。

### P2：据点压力可视化

让玩家更清楚地看到据点是谁在抢。

建议在 HUD 或 Canvas 上强化：

| 信息 | 表现 |
| --- | --- |
| 当前占领方 | 据点外圈颜色更明显 |
| 当前争夺方 | 边缘细线或轻脉冲 |
| 圈内压力 | 用三方短条或小数字显示 |
| 僵持 | 中性色，不涨分提示 |

第一版优先 Canvas，不要在每帧频繁刷新复杂 ArkUI 组件。HUD 可以继续使用已有 `controlZone` 快照。

### P3：道具生成靠近战场

当前道具全场随机，手机端可能导致单位分散。建议把 `团队占点` 的道具生成策略调成“据点外围优先”：

| 项 | 建议 |
| --- | --- |
| 生成区域 | 据点外圈附近，避开据点核心 |
| 边缘限制 | 不刷极端角落 |
| 最大数量 | 仍保持 `6` |
| 首次生成 | `3s` 保持，方便开局看到道具 |
| 类型轮换 | 保持加速、护盾、分裂轮换，便于真机观察 |

这样道具会推动单位回到中心附近，而不是把战局拉散。

### P4：黑洞位置节奏

黑洞在团队占点中应该是“扰乱中心路线”，不是“清掉中心据点”。

建议：

| 项 | 建议 |
| --- | --- |
| 首次出现 | 保持 `9s..16s`，不抢开局据点焦点 |
| 生成位置 | 避开据点核心和复活点 |
| 理想位置 | 据点外围路线附近 |
| 移动端上限 | 保留当前短边动态上限 |
| 是否更早出现 | 暂不提前，除非真机反馈前半局太平 |

和 `小心叛徒` 不同，团队占点不需要黑洞提前。开局应该先让玩家理解据点目标。

### P5：三方独立占点进度

这是中风险改动，不建议马上做，但要留口子。

当前：

```text
owner + score
```

备选：

```text
zoneScores: rock / scissors / paper
```

优点：

- 每个阵营都有自己的占点成果。
- 反抢不会继承对方进度。
- 时间结束时按最高占点分更直观。

风险：

- 类型、HUD、Canvas、结算和测试都要一起改。
- 可能偏离原版单据点 `owner + score` 手感。
- 需要先确认当前单分数模型在真机上是否真的不好理解。

建议等 P1 到 P4 真机反馈后再决定。

## 不建议本轮做的事

- 不做多个据点。手机竖屏空间不够，AI 分兵和 HUD 解释都会变复杂。
- 不把所有机制改成战斗中开关。机制开关仍停留在开局配置阶段。
- 不提高总人数上限。当前性能和可读性都不适合继续堆单位。
- 不让黑洞开局立刻出现。团队占点开局应先突出中心据点。
- 不把援军做成无限复活。弱势方能回场，但仍要允许对局结束。

## 推荐实施顺序

第一轮建议只做 P1 + P2 的轻量版本：

1. 开局前 `4s` 提高占点意图，但保留天敌逃离优先级。
2. 据点 Canvas 加开局目标脉冲和更清晰的争夺状态。
3. 性能日志继续观察 `zoneMs`、`targetMs`、`collisionMs`、`fps`、`maxInterval`。
4. 补 `RpsBattleZoneSystem` 或引擎占点意图相关 Hypium 测试。
5. 写测试报告，真机看开局是否更像抢点。

第二轮再做 P3：

1. 道具生成改成据点外围优先。
2. 验证道具是否更容易被看到和拾取。
3. 观察是否会导致中心过度拥挤。

第三轮再评估 P4/P5：

1. 黑洞是否需要更明确地避开据点核心。
2. 是否需要三方独立占点进度。

## 验收标准

真机验收重点：

- 手机标准局开局 `5s` 内，中心据点争夺感明显。
- 据点分数变化和当前占领方更容易看懂。
- 黑洞、道具、援军不会抢走据点主线。
- 开启全部机制时仍能看出“这是团队占点”，不是普通乱斗加事件。
- 关闭黑洞、绝地求生、道具后，对应机制不出现，中心据点仍正常。
- `RpsBattlePerf` 中 `fps` 接近当前水平，`maxInterval` 不出现明显恶化。

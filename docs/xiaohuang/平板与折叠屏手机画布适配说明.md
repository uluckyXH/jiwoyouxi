# 平板与折叠屏手机画布适配说明

更新时间：2026-07-24

## 目标

在真正为平板设计独立布局前，让已接入页面在平板、展开态折叠屏和高尺寸分屏窗口中保持接近手机的视觉节奏，避免卡片、棋盘和 Canvas 可玩区被直接拉满。

本次方案不改变游戏玩法，也不缩放触摸坐标与画面坐标的关系；页面内部仍以自身 `pageWidth` 和 `pageHeight` 完成布局。

## 适用范围

已接入页面：

- App 大厅：`HubPage.ets`，包括首页、收藏、成就和设置 Tab。
- 鸡窝水果摊：`SuikaPage.ets`。
- 扫雷：`MinesweeperPage.ets`。
- 俄罗斯方块：`TetrisPage.ets`。
- 鸡窝 2048：`Chicken2048Page.ets`。
- 空当接龙：`FreecellPage.ets`。

暂未纳入本次统一画布：`rps`、`rpsBattle`、`eggSling`、`whiteTiles`。其中弹射鸡蛋仍有独立横屏逻辑，不能默认套用本方案。

## 尺寸规则

共享逻辑位于 `entry/src/main/ets/design/CoopViewport.ets`。

| 窗口情况 | 画布尺寸 |
| --- | --- |
| 宽度不超过 `430vp`，且高度不超过 `932vp` | 保持实际可用宽高，不改变普通手机体验。 |
| 宽度或高度超出普通手机范围 | 使用 `390vp x 844vp` 的标准手机画布，并在可用空间不足时等比缩小。 |

大屏画布通过外层 `Stack` 居中，页面内部只接收成对计算后的宽高。因此不会出现“宽度已限制、高度仍被拉长”导致的比例偏差。

`390vp x 760vp` 仅是部分旧游戏组件在获得真实区域前的初始化值，不作为大屏固定画布基准。使用它会与常见手机实际画面产生高度差。

## 大厅处理

- 大厅内容和底部 Tab 均放入居中的手机画布。
- 外层背景与大厅内容背景保持一致。
- `Scroll` 使用 `BarState.Off` 隐藏滚动条，保留手势滚动，避免折叠屏上出现突兀的长滚动条。

## 鸡窝水果摊处理

- 水果摊与其他已接入游戏使用相同的手机画布规则。
- 画布外层使用 `CoopTheme.appSurface`（`#fff4e4`），与大厅浅色背景一致。
- 游戏场景内部的加载兜底使用木屋场景棕色 `#3d2413`，避免图片加载时露出蓝色底色。
- Canvas、背景图、投放区和物理边界共享同一组 `pageWidth`、`pageHeight`，保证大屏中仍保持手机画面的相对位置。

## 验收与限制

- 已完成代码静态检查：`git diff --check` 通过。
- 当前折叠屏页面视觉已人工确认无明显布局偏差，主页滚动条已隐藏，水果摊外围背景已与大厅统一。
- 本机无法完成完整构建验证：`./scripts/build-app.sh --stacktrace` 依赖的 DevEco `hvigorw` 文件缺失。
- 本次不包含全局方向锁定。相关方向策略和弹射鸡蛋的横屏冲突见《全局竖屏锁定与平板适配建议》。

## 后续维护

新增需要在平板/折叠屏中保持手机画面的页面时，应复用 `CoopViewport.frame(width, height)`，并使用返回的宽高同时更新页面状态。不要分别限制宽度和高度。

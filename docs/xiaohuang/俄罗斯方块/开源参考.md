## 为什么选它

- 完成度高，有完整 UI、音效、暂停、分数、等级、next/hold 等模块。
- TypeScript 项目，规则层和 UI 层比纯 JavaScript demo 更清晰。
- README 里明确说明它重写过游戏循环和核心逻辑结构，适合作为“怎么组织俄罗斯方块代码”的参考。
- MIT 许可证，后续如果借鉴思路或少量代码，合规风险低于无许可证项目。

## 对当前 HarmonyOS 项目的参考方式

不要直接照搬 Angular 组件层。当前项目是 ArkTS / ArkUI，最值得参考的是：

- 规则层如何拆分成独立类和工具。
- piece shape 是否预定义为矩阵，避免渲染期反复旋转计算。
- 游戏状态如何集中推进，而不是分散在 UI build 函数里派生。
- next、hold、score、level 这些 UI 模块如何分层。
- 按键反馈、暂停/结算、next/hold 信息组织等交互细节。

不参考它的复古掌机视觉。鸡窝游戏里的俄罗斯方块应该继续走当前项目的轻量、清爽、偏现代的视觉方向，可以保留鸡窝主题和 HarmonyOS 原生质感，不做像素掌机复刻。

当前性能优化仍建议按 `性能优化分析.md` 先做阶段 1：

1. 去掉棋盘 key 里的 `tick`。
2. 预计算可见棋盘。
3. 避免每个格子重复计算幽灵落点。
4. 硬降改为一次性提交状态。

如果阶段 1 后真机仍卡，再考虑把棋盘改成 Canvas。

# 首页浅深色主题设计

更新时间：2026-06-10

## 背景

当前首页和参考稿之间最大的差距不是动画，而是首页视觉系统还不统一：背景、卡片、底部 Tab 栏、文字颜色和深色模式没有形成一套可切换的主题。

本阶段先只处理 App 首页大厅，不处理各游戏内部页面。游戏页仍由各游戏自己的主题和开发节奏决定。

## 目标

- 统一首页纯色背景、卡片、文字、线条、阴影和底部 Tab 栏。
- 支持浅色和深色两种外观。
- 在首页头部和设置页提供“深色模式”开关。
- 外观选择持久化到本地快照。
- 不影响游戏页玩法、布局和动效。

## 范围

覆盖：

- `HubPage.ets` 首页、收藏、成就、设置。
- 首页纯色背景。
- 顶部问候区。
- 游戏卡片。
- 数据统计卡片。
- 收藏列表。
- 成就列表。
- 设置项。
- 满宽嵌入式底部 Tab 栏。

不覆盖：

- 各游戏页。
- 游戏内部弹窗。
- 游戏内部深色模式。
- 系统级自动深浅色跟随。

## 实现方案

### 安全区和系统栏

应用入口继续使用全屏/沉浸式窗口布局：

- `setImmersiveModeEnabledState(true)`：开启主窗口沉浸模式。
- `setWindowLayoutFullScreen(true)`：允许页面内容延伸到系统栏区域。

首页内部自己处理可读区域：

- 顶部内容从固定 56vp 开始，避免问候语贴近状态栏、挖孔和胶囊区。
- 底部 Tab 高度为 96vp，其中底部 24vp 用于承接手势导航条。
- 底部 Tab 使用 `expandSafeArea([SafeAreaType.SYSTEM], [SafeAreaEdge.BOTTOM])`，让导航背景延伸到底部系统安全区。
- Tab 内部左右边距维持 20vp，和页面内容左右对齐。

### 状态存储

在 `CoopAppSnapshot` 中增加：

```ts
appearanceMode: string
```

当前取值：

- `light`
- `dark`

`CoopStorage` 读取旧数据时会兜底为 `light`，避免老快照缺字段导致异常。

### 首页主题

`HubPage.ets` 内部根据 `snapshot.appearanceMode` 派生纯色 token：

- 页面背景色。
- 主文字颜色。
- 次级文字颜色。
- 卡片底色。
- 边框线。
- 阴影。
- 底部 Tab 背景。
- Tab 选中和未选中颜色。

当前先在 `HubPage.ets` 内部收敛这些 palette 方法。后续如果首页视觉继续扩大，再考虑抽到 `design/CoopHomeTheme.ets`。

首页不再使用大面积渐变背景。底部 Tab 不再做悬浮胶囊，而是贴底、满宽、内部左右边距与页面内容保持 20vp 对齐。

### 切换入口

首页头部右上角提供月亮/太阳图标按钮，用于快速切换浅色和深色。这个入口用于日常验证和高频切换，避免用户必须进入设置页才能看到主题效果。

设置页同步保留：

设置页新增：

```text
深色模式：开 / 关
```

切换后通过 `Index.ets` 更新 `CoopAppSnapshot`，并走 `CoopStorage` 持久化。

## 验收点

- 默认进入首页为浅色。
- 设置页打开“深色模式”后，首页纯色背景、卡片和 Tab 栏切换到深色。
- 切换 Tab 后深色样式保持一致。
- 返回大厅后深色样式保持一致。
- 重启 App 后外观选择仍保留。
- 进入任意游戏后，游戏页面不被首页深色样式影响。
- 小屏下顶部问候语不贴状态栏/挖孔区。
- 小屏下底部 Tab 栏贴底满宽，内部图标和页面内容左右对齐，背景承接手势导航条。

## 后续优化

- 如果需要跟随系统深浅色，再增加 `system` 模式。
- 如果首页样式继续复杂化，把 palette 方法抽到 `design/CoopHomeTheme.ets`。
- 如果游戏页成熟后需要深色支持，再按游戏单独设计，不强行继承首页主题。

# 合成大西瓜水果素材切图清单

## 目标

本文件先只定义「鸡窝水果摊」的水果素材。背景、木箱、按钮、弹窗、粒子后续再单独补清单。

当前玩法内部保留 0 到 10 共 11 个等级，素材按等级一一替换。所有水果都要保持圆滚滚、玩具感、半眯眼表情，和鸡窝游戏整体气质统一。

## 通用导出要求

- 格式：PNG。
- 背景：透明。
- 色彩：sRGB。
- 命名：全小写英文 + 等级编号，避免中文文件名。
- 画布：建议统一 `512x512 px`。
- 安全区：主体不要贴边，四周至少留 `40 px` 透明边距。
- 主体占比：水果主体建议占画布宽高的 `76% ~ 84%`，鸡冠、叶子、高光可以略微超出主体，但不要超出安全区。
- 阴影：水果本体可以有轻微内阴影和高光，不建议自带大投影。游戏里会根据需要统一处理阴影。
- 文字：不要把名称、等级、分数烘焙进图片。
- 表情：每个水果都带半眯眼表情；眼睛风格统一，不要每个水果一套表情系统。
- 风格：圆润、软弹、偏玩具水果，不要写实水果摄影风。

## 尺寸策略

虽然游戏里不同等级显示大小不同，切图仍建议全部使用统一 `512x512 px` 画布。

原因：

- 代码可以按等级统一缩放，适配不同机型更稳定。
- 后续替换素材时不需要为每个等级单独写宽高。
- 碰撞半径仍由代码控制，不受图片实际像素尺寸影响。

如果需要更高清，也可以导出 `1024x1024 px` 原图存档，但进入工程的运行素材建议先使用 `512x512 px`，控制包体。

## 视觉安全区说明

每张图可以按下面结构设计：

```text
512x512 透明画布
┌────────────────────┐
│      40px 安全边距  │
│  ┌──────────────┐  │
│  │   主体水果    │  │
│  │  约 390-450px │  │
│  └──────────────┘  │
│      40px 安全边距  │
└────────────────────┘
```

注意：

- 等级小不代表图片主体要很小。实际游戏里的大小由代码缩放。
- `fruit_09_rooster_watermelon.png` 的鸡冠要算进安全区，不能被画布裁掉。
- `fruit_10_giant_chicken_watermelon.png` 可以更有识别度，但不要做成复杂场景图。

## 表情规范

统一表情建议：

- 眼睛：两条短弧线或短横线，半眯眼。
- 嘴巴：可选。小等级可以不画嘴，大等级可以加一点点嫌弃嘴角。
- 位置：眼睛位于水果中心线上方约 `8% ~ 15%`。
- 颜色：深棕、深灰或接近黑色，但透明度不要过重。
- 不要使用夸张大眼、卡通泪光或复杂五官，避免和主角鸡风格冲突。

## 验收标准

切图完成后逐张检查：

- 文件名和等级对应正确。
- PNG 是透明背景。
- 画布尺寸一致。
- 主体没有贴边、裁切、阴影被截断。
- 半眯眼表情在缩小显示后仍能看清。
- 11 个水果从小到大有明显视觉成长感。
- `fruit_09` 和 `fruit_10` 有鸡窝/鸡冠特色，能和普通西瓜区分。

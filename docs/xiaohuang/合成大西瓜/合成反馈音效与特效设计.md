# 合成大西瓜合成反馈音效与特效设计

## 目标

合成反馈对齐参考项目的体验方向，但使用鸿蒙原生 ArkTS、Canvas 和我们自己的素材/音频实现。

当前按协作开发需要，先把参考项目里的合成音效拿到本项目作为过渡资源，同时复用它的触发逻辑：

- 普通合成：播放短促合成音效，出现果汁/星星/鸡毛粒子，显示分数浮动。
- 最终合成：在普通合成反馈基础上，增加庆祝音效、彩带/星星扩散和最终文案。
- 失败结算不混入合成音效，保持事件边界清晰。

## 音效是什么

音效是独立音频文件，放在 `entry/src/main/resources/rawfile/` 下，由鸿蒙原生音频能力加载播放。

推荐格式：

| 类型 | 建议 |
| --- | --- |
| 文件格式 | `mp3` 优先，短音效也可以用 `wav` |
| 时长 | 普通合成 0.2 到 0.6 秒，最终合成 0.8 到 1.5 秒 |
| 音量 | 文件本身不要压得太响，代码侧再按事件设置音量 |
| 声音风格 | 软弹、轻快、带一点玩具感，不做尖锐爆炸 |

推荐目录：

```text
entry/src/main/resources/rawfile/games/suika/audio/
```

推荐文件名：

```text
merge_pop.mp3        普通合成，短促“啵/咚”
merge_splash.mp3     果汁飞散，轻微湿润颗粒感
merge_final.mp3      最终大西瓜合成，庆祝感
land_soft.mp3        可选，落地软弹音
warning_line.mp3     可选，接近失败线警示
```

## 参考项目逻辑

本地参考项目路径：

```text
reference-projects/liyupi-daxigua
```

关键触发点：

- `onBeginContact` 里判断同等级水果碰撞后合成。
- 普通合成会调用 `createFruitL(...)`。
- `createFruitL(...)` 内会播放两段音效，并生成果汁碎片、果汁粒子和中心扩散效果。
- 最终大西瓜合成额外播放庆祝音效，并触发彩带/星星类效果。

参考音效资源名能在 `src/settings.js` 里看到：


参考项目使用方式是 `AudioManager.Play(index, loop, volume)`，本质上也是事件触发音频片段。

我们先对齐触发时机和反馈层级；当前音频文件是过渡版本，后续可替换为原创音效。

## 我们的事件设计

在合成大西瓜里定义独立音效事件，不复用猜拳游戏已有的 `attack/warning/snap` 等 key。

建议事件：

| 事件 key | 触发时机 | 音频文件 | 说明 |
| --- | --- | --- | --- |
| `suika_merge_pop` | 任意普通合成 | `games/suika/audio/merge_pop.mp3` | 主合成音 |
| `suika_merge_splash` | 任意普通合成 | `games/suika/audio/merge_splash.mp3` | 果汁/粒子辅助音 |
| `suika_merge_final` | 等级 9 合成等级 10 | `games/suika/audio/merge_final.mp3` | 终局庆祝音 |
| `suika_land_soft` | 水果首次明显落地 | `games/suika/audio/land_soft.mp3` | 可选，避免声音太密 |
| `suika_warning_line` | 持续靠近失败线 | `games/suika/audio/warning_line.mp3` | 可选，需冷却 |

第一阶段已接：

```text
suika_merge_pop
suika_merge_splash
suika_merge_final
```

原因：合成是高频但可控事件，最容易验证；普通合成增加短冷却，避免连锁合成时音效过密。

## 视觉特效设计

当前阶段不等正式粒子素材，先用 Canvas 程序绘制占位。

普通合成：

- 合成点出现一圈短暂扩散光圈。
- 8 到 14 个小粒子向外飞散。
- 粒子颜色取当前水果颜色、白色、暖黄色。
- `+N` 分数文字继续上浮。
- 新水果可以短暂从 0.92 放大到 1.05 再回落，后续再做。

最终合成：

- 触发普通合成粒子。
- 屏幕中心出现“巨型鸡窝西瓜！”。
- 增加更多星星/鸡毛方向粒子。
- 播放最终合成音效。

后续有正式素材后，可替换为：

```text
entry/src/main/resources/rawfile/games/suika/effects/
  particle_star.png

  particle_feather.png
  ring_merge.png
```

## 接入方案

### 音频服务

当前项目已有：

```text
entry/src/main/ets/services/AudioService.ets
```

建议改造为支持更多 sound key：

- 扩展 `SoundKey`，加入 `suika_merge_pop`、`suika_merge_final`。
- `SOUND_ASSETS` 增加对应 rawfile 路径。
- `SuikaPage` 自己持有 `NativeAudioService`，进入页面时初始化，离开页面时释放。
- 合成发生时调用 `this.audioService.play('suika_merge_pop')`。
- 最终合成时额外调用 `this.audioService.play('suika_merge_final')`。

注意：

- 不建议每个水果碰撞都播放音效，只在真正合成时播放。
- 普通合成音效要做短冷却，避免一帧连锁合成时声音重叠太吵。
- 如果音频文件不存在，音频服务应只打印日志，不影响游戏逻辑。

### Canvas 特效

建议在 `SuikaPage.ets` 内先加轻量结构，后续再拆文件：

```text
MergeParticle
  x
  y
  vx
  vy
  radius
  color
  bornAt
  duration

MergeRing
  x
  y
  bornAt
  duration
  color
```

触发点：

```text
resolveCollisions(now)
  if result.didMerge:
    score += result.scoreDelta
    spawnMergeEffects(result.mergeX, result.mergeY, result.mergeLevel)
    playMergeSound(result.isFinalMerge)
```

绘制顺序：

```text
背景
容器
水果阴影
水果
合成粒子/光圈
分数浮动
最终合成文案
```

## 第一阶段实现清单

1. 新建 `entry/src/main/resources/rawfile/games/suika/audio/`。
2. 音频文件已加入：
   - `merge_pop.mp3`
   - `merge_splash.mp3`
   - `merge_final.mp3`
3. 扩展 `NativeAudioService` 的 sound key。
4. `SuikaPage` 初始化/释放音频服务。
5. 普通合成播放 `suika_merge_pop`。
6. 最终合成播放 `suika_merge_final`。
7. Canvas 增加程序粒子和扩散光圈。
8. 自动化测试保持规则逻辑不依赖音频和视觉特效。
9. 构建验证通过后，再真机/模拟器确认音量和触发频率。

## 验收标准

- 合成时能听到短促反馈音。
- 最终合成与普通合成有明显区别。
- 连锁合成时声音不刺耳、不堆叠成噪音。
- 没有音频文件时游戏仍能正常运行。
- 粒子不会遮挡水果主体和失败线判断。
- 规则测试不依赖音频资源，音频失败不影响玩法结果。

# 合成大西瓜水果素材离线边缘处理方案

## 背景

当前「鸡窝水果摊」的水果素材已经是透明 PNG，但木板预览区域会把水果固定放大展示。放大后，部分素材边缘能看到抠图黑边、锯齿或透明边缘脏色。

这类问题不适合继续用运行时代码参数硬调。水果不是规则圆形，叶子、鸡冠、表情和水果主体都会形成不规则轮廓。正确方向是基于 PNG 的 alpha 通道计算真实轮廓，离线生成处理后的预览素材，运行时直接使用。

## 目标

- 木板预览水果使用固定展示大小，保证摆在木板上时稳定、精致。
- 游戏内落下后的水果继续按各自设计半径显示，保留大小成长关系和物理碰撞手感。
- 预览素材边缘通过离线 alpha 轮廓处理解决黑边、锯齿和透明脏色。
- 运行时不做重型像素计算，避免影响启动和游戏帧率。
- 预览资源和正式游戏资源分开，避免预览优化影响真实玩法区。

## 不做的事

- 不按规则圆形裁切水果。水果有叶子、鸡冠等不规则区域，圆形算法会误伤素材。
- 不在每一帧动态分析边缘。
- 不用运行时代码给所有水果强行描边。
- 不改变真实下落水果的物理半径和合成规则。

## 资源分层

建议保留两套运行资源：

```text
entry/src/main/resources/rawfile/games/suika/fruits/
  fruit_00_grape.png
  fruit_01_blueberry.png
  ...

entry/src/main/resources/rawfile/games/suika/fruits/preview/
  fruit_00_grape_preview.png
  fruit_01_blueberry_preview.png
  ...
```

用途：

- `fruits/fruit_xx.png`：真实游戏水果，用于下落、碰撞、合成。
- `fruits/preview/fruit_xx_preview.png`：木板预览水果，用于投放前展示。

后续如果要保留原始未处理素材，可以再增加：

```text
assets/source/suika/fruits/
```

但进入 App 包体的 rawfile 里只放运行时需要的图片。

## 离线处理方案

离线脚本读取每张 PNG 的 RGBA 像素，基于 alpha 通道处理边缘。

### 1. 识别有效像素

定义：

- `alpha >= 16`：视为水果或装饰的一部分。
- `alpha < 16`：视为透明背景。

这个阈值可在脚本里调整。阈值太低会把透明噪点算进去，太高可能吃掉半透明抗锯齿边。

### 2. 计算真实轮廓

遍历所有有效像素，如果它周围 8 个方向存在透明像素，就认为它是轮廓像素。

这样得到的是 PNG 真实形状，而不是规则圆：

- 水果主体会被算入。
- 叶子会被算入。
- 鸡冠会被算入。
- 其他不规则装饰也会被算入。

### 3. 做透明边缘补色

黑边通常来自透明像素的 RGB 是黑色或深色。缩放时 Canvas 会采样透明边缘 RGB，导致出现脏边。

处理方式：

- 对轮廓附近一定范围内的透明或半透明像素，寻找最近的不透明水果像素。
- 把这些透明边缘像素的 RGB 替换成附近水果颜色。
- 保持原 alpha 不变。

关键点：

- 只修 RGB，不强行改 alpha。
- 不改变素材外形。
- 不扩大水果尺寸。
- 不破坏原本的透明抗锯齿。

这个处理常见名称是：

```text
alpha bleed
transparent pixel solidify
edge color padding
```

### 4. 生成预览专用图

预览图可以统一输出 `512x512`，但内容需要符合木板展示：

- 主体看起来尺寸统一。
- 边缘已经做过补色。
- 仍然保留透明背景。
- 不自带大投影。
- 不烘焙木板背景。

## 运行时使用方式

`FRUITS` 后续可以扩展字段：

```ts
{
  name: '橘子',
  radius: 21,
  color: '#f97316',
  image: 'games/suika/fruits/fruit_02_orange.png',
  previewImage: 'games/suika/fruits/preview/fruit_02_orange_preview.png'
}
```

绘制规则：

- 木板预览：使用 `previewImage`，固定预览半径。
- 下落水果：使用 `image`，按 `radius` 绘制。

这样木板预览和真实游戏区互不干扰。

## 尺寸策略

木板预览：

- 所有水果固定显示同一视觉大小。
- 由 `SUIKA_DROP_PREVIEW_FIXED_RADIUS` 和 `SUIKA_DROP_PREVIEW_SCALE` 控制。
- 只用于告诉玩家当前手上水果是什么。

真实下落：

- 每种水果按 `FRUITS[level].radius` 显示。
- 小葡萄小，西瓜大。
- 这个大小参与碰撞、堆叠、失败线判断和合成节奏。

两者不能混用。预览图的精修不应该改变真实玩法区的水果大小。

## 脚本建议

建议新增脚本：

```text
scripts/generate-suika-preview-assets.mjs
```

建议能力：

- 扫描 `entry/src/main/resources/rawfile/games/suika/fruits/fruit_*.png`。
- 跳过 `preview/` 目录，避免重复处理。
- 读取 PNG RGBA 像素。
- 执行 alpha bleed。
- 输出到 `entry/src/main/resources/rawfile/games/suika/fruits/preview/`。
- 保持文件名对应，例如：

```text
fruit_02_orange.png -> preview/fruit_02_orange_preview.png
```

生成后的 `preview` 图片需要进入版本管理，因为 App 打包时需要直接读取这些资源。

## 验收标准

- 木板预览水果边缘没有明显黑边或脏色。
- 木板预览水果没有因为边缘处理变大。
- 木板预览水果固定大小展示。
- 下落后的水果仍按各自半径显示。
- 合成、碰撞和失败判断不受预览图影响。
- 真机和模拟器上都没有明显锯齿、描边感或发光感。

## 后续实施顺序

1. 新增离线处理脚本。
2. 生成 `fruits/preview/` 预览图。
3. 扩展 `FRUITS` 增加 `previewImage`。
4. 木板预览改用 `previewImage`。
5. 删除当前运行时木板预览边缘修补参数，避免重复处理。
6. 构建并真机检查木板预览和真实下落效果。

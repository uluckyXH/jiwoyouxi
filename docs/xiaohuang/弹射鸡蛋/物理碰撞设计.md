# 弹射鸡蛋物理碰撞设计

## 当前目标

当前阶段先不做正式关卡系统，也不做关卡选择页。

这一阶段只打磨一个“测试场景”里的核心物理体验：

- 弹弓拉开要有劲。
- 鸡蛋飞行轨迹要可预期。
- 撞到地面、边界、障碍、目标都要有清楚反馈。
- 障碍物要从“静态方块”逐步变成“能被撞坏、撞动、撞碎”的游戏对象。
- 后续素材替换时，不需要推翻碰撞规则。

## 技术边界

当前实现基于：

```text
ArkTS + ArkUI + Canvas + 轻量手写物理
```

这个方案适合做“简易愤怒的小鸟式”的手感，但不适合一开始追求完整刚体引擎。

适合先做：

- 抛物线飞行。
- 圆形鸡蛋碰撞。
- 矩形障碍碰撞。
- 圆形目标碰撞。
- 星星收集碰撞。
- 地面、墙体、顶部边界反弹。
- 障碍耐久和破碎状态。
- 简单冲量反馈。
- 简单粒子和屏幕反馈。
- 不同鸡蛋类型的差异参数。

暂时不做：

- 多个障碍之间的真实刚体挤压。
- 木板被撞后真实旋转倒塌。
- 连锁堆叠坍塌。
- 复杂摩擦、关节、约束。
- 高精度连续碰撞检测。

原因是这些会把项目推向物理引擎级别，开发成本高，而且第一阶段不一定能明显提升好玩程度。

## 已有基础

当前代码已经具备：

- 拖拽蓄力。
- 鸡蛋按速度和重力飞行。
- 鸡蛋和目标的圆形碰撞。
- 鸡蛋和矩形障碍的碰撞。
- 左右边界、顶部边界、地面边界。
- 边界反弹和速度衰减。
- 鸡蛋低速落地后停止。
- 轨迹预览。
- 视觉拖拽距离和物理发射力度分离。

所以后续重点不是从零实现物理，而是把碰撞反馈做得更像游戏。

## 第一批可做项

### 1. 碰撞类型统一

把碰撞体分成几类：

- `egg`：鸡蛋，圆形碰撞体。
- `target`：目标，圆形或矩形碰撞体。
- `block`：障碍，矩形碰撞体。
- `star`：星星，圆形收集碰撞体。
- `boundary`：世界边界。

第一版只需要圆形和轴对齐矩形，不做旋转矩形。

验收标准：

- 碰撞判断入口统一。
- 后续新增星星或新蛋种，不需要复制一套碰撞代码。

### 2. 鸡蛋撞障碍反馈

当前障碍只是扣血，反馈还弱。

可以增加：

- 障碍被撞后短暂闪白或变浅。
- 障碍根据剩余耐久显示不同颜色或裂纹占位。
- 鸡蛋撞障碍后根据撞击方向反弹。
- 撞击速度越大，扣血越高。
- 撞击速度太低时，只轻微反弹，不扣血。

第一版规则：

```text
impactSpeed = abs(vx) + abs(vy)
impactSpeed < 2.5：只反弹，不扣血
2.5 <= impactSpeed < 6：扣 1 点耐久
impactSpeed >= 6：扣 2 点耐久
```

验收标准：

- 轻轻碰到障碍不会像重击一样碎。
- 快速撞击有明显破坏感。
- 障碍状态变化能被玩家看懂。

### 3. 鸡蛋撞目标反馈

目标被命中后不应该只是立刻消失。

可以增加：

- 命中后短暂缩放或闪烁。
- 目标进入 `hit` 状态，再延迟消失。
- 命中位置生成少量碎片或星星粒子。
- 鸡蛋命中后速度明显衰减，但不一定立刻停止。

第一版规则：

- 普通目标命中一次即击破。
- 击破时加分。
- 击破后保留 8 到 12 帧动画再移除。

验收标准：

- 玩家能看出“我打中了”。
- 鸡蛋命中目标后不会像穿透一样没有重量。

### 4. 边界碰撞反馈

当前边界已经能反弹，但反馈可以更明确。

可以增加：

- 撞左/右边界时边界线闪一下。
- 撞地面时产生轻微尘土。
- 撞顶部时降低反弹系数，避免无限弹。
- 连续撞边界时加快速度衰减。

验收标准：

- 鸡蛋不会无反馈飞出画面。
- 撞墙、撞地、撞顶部的手感有区别。

### 5. 星星收集碰撞

星星先不作为通关条件，只作为加分项。

可以增加：

- 星星用圆形碰撞体。
- 鸡蛋经过星星半径内即收集。
- 收集后星星播放缩放或淡出。
- 收集数量用于后续结算星级。

第一版规则：

```text
star.radius = 14
egg-star 碰撞后 collected = true
score += 100
```

验收标准：

- 星星不会影响鸡蛋飞行。
- 星星收集反馈清楚。

### 6. 鸡蛋类型预留

先不急着全做，但碰撞代码要能支持不同蛋种。

建议预留：

- 普通蛋：默认质量、默认反弹。
- 弹弹蛋：边界和障碍反弹更强。
- 爆裂蛋：命中后范围伤害。
- 重量蛋：重力更大，撞击伤害更高，反弹更弱。
- 分裂蛋：飞行中触发分裂，生成小蛋。

第一阶段可以先做普通蛋参数结构，不展示选择 UI。

验收标准：

- 普通蛋仍保持当前手感。
- 代码里已经能通过 `egg.kind` 或类似字段切换物理参数。

## 第二批可做项

### 1. 障碍受力偏移

不做真实刚体，但可以做“假位移”。

规则：

- 障碍被撞后按撞击方向偏移几像素。
- 偏移后缓慢回弹，或者保持轻微歪斜。
- 耐久归零后消失或变成碎片。

这样能提升撞击感，又不需要完整物理引擎。

### 2. 简单碎片系统

障碍或目标被击破时生成临时碎片。

碎片只做视觉，不参与碰撞：

- 初速度随机。
- 受重力影响。
- 透明度逐渐降低。
- 0.4 到 0.8 秒后移除。

### 3. 命中暂停和轻微震动

重击时可以做很短的 hit stop：

- 命中一瞬间暂停 2 到 4 帧。
- 画面轻微抖动。
- 重击才触发，普通碰撞不触发。

这个会明显增强手感，但要控制幅度，不要影响操作。

## 暂缓项

以下内容先不做：

- 正式关卡选择。
- 多关卡解锁。
- 复杂星级结算。
- 多蛋种 UI。
- 障碍真实倒塌。
- 多物体真实碰撞堆叠。

等基础碰撞和反馈稳定后，再进入关卡配置。

## 建议实现顺序

推荐按这个顺序做：

1. 新增碰撞结果结构，统一描述命中对象、命中速度、命中方向。
2. 调整障碍扣血规则，让撞击速度影响伤害。
3. 给障碍增加 `hitFrames` 或 `damageLevel`，先做视觉反馈。
4. 给目标增加 `hitFrames`，命中后延迟消失。
5. 增加星星类型和收集碰撞，但先不做关卡 UI。
6. 增加简单粒子数组，用于目标击破、障碍破碎、落地尘土。
7. 把普通蛋参数抽成配置，为弹弹蛋、重量蛋预留。
8. 再根据测试手感微调反弹、衰减和伤害阈值。

## 第一批实现口径

本次先实现第 1 和第 2 类反馈，不做碎片和尘土。

已落地范围：

- 新增独立碰撞模块，页面不直接承载障碍伤害和倒塌规则。
- 障碍命中后有短暂偏移、轻微旋转、闪色和裂纹占位。
- 障碍伤害根据撞击速度分级，轻碰只反弹，重击扣更多耐久。
- 障碍被打坏后进入动态下落状态，受重力、地面反弹和摩擦衰减影响。
- 同组支撑或横梁被打坏后，剩余同组障碍会触发假倒塌。

暂不实现：

- 碎片粒子。
- 落地尘土。
- 屏幕震动。
- 真实刚体堆叠。

这保证当前阶段先解决“障碍太僵硬”的问题，同时不扩大到完整物理引擎。

## 代码落点

建议继续保持当前拆分方向：

```text
entry/src/main/ets/games/eggSling/
  EggSlingPage.ets        页面、Canvas、输入、主循环
  EggSlingTypes.ets       类型定义
  EggSlingPhysics.ets     速度、重力、边界、碰撞基础函数
```

后续可以新增：

```text
  EggSlingCollision.ets   碰撞检测和碰撞结果
  EggSlingEffects.ets     粒子、命中闪烁、短暂停顿参数
  EggSlingEggs.ets        鸡蛋类型参数
```

拆分原则：

- 物理计算不要写进 UI Builder。
- 碰撞判断不要散落在多个绘制函数里。
- Canvas 绘制只根据实体状态画，不负责改规则。
- 第一阶段不要过度拆文件，先把行为跑通。

## 验收标准

这一阶段完成后，应该满足：

- 鸡蛋撞不同对象有不同反馈。
- 轻撞和重撞结果不同。
- 障碍可以被逐步打坏。
- 目标被击中有命中反馈，而不是瞬间消失。
- 星星可以被收集，但不影响通关。
- 鸡蛋不会飞出画面或无限弹跳。
- 当前测试场景在不同设备尺寸下仍能正常玩。

达到这些后，再开始做固定关卡配置。

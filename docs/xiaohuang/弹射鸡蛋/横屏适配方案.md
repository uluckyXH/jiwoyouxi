# 弹射鸡蛋横屏方案

## 背景

弹射鸡蛋的核心玩法是横向抛射：左侧弹弓发射鸡蛋，右侧摆放目标、星星和障碍。竖屏空间偏窄，会导致：

- 发射距离太短。
- 抛物线不明显。
- 目标和障碍只能堆在下方，关卡空间不够。
- 手指拖拽区域容易挡住弹弓和鸡蛋。

所以弹射鸡蛋应该按横屏优先设计。

## 鸿蒙是否支持横屏检测和窗口方向

支持。

当前项目里，`EggSlingPage.ets` 使用两类能力：

- `display.getDefaultDisplaySync().orientation`：读取当前显示方向，用于首次竖屏提示。
- `window.Window#setPreferredOrientation(...)`：进入弹射鸡蛋时请求主窗口横屏，退出时恢复默认方向。

页面根容器仍通过 `.onAreaChange(...)` 获取页面尺寸：

```text
pageWidth
pageHeight
```

尺寸兜底判断：

```text
pageWidth > pageHeight  => 横屏
pageWidth <= pageHeight => 竖屏
```

这样既不需要把整个 `EntryAbility` 写死成横屏，也能让弹射鸡蛋进入后真正切到横屏窗口。

## 当前工程情况

当前入口：

```text
entry/src/main/ets/entryability/EntryAbility.ets
```

它做了全屏窗口：

```text
mainWindow.setWindowLayoutFullScreen(true)
```

当前没有在 `entry/src/main/module.json5` 里为 `EntryAbility` 单独配置横屏。

小游戏路由在：

```text
entry/src/main/ets/pages/Index.ets
```

这意味着如果直接把整个 Ability 配成横屏，可能会影响大厅、扫雷和其他小游戏。弹射鸡蛋需要横屏，但其他游戏未必需要。

## 推荐方案

第一阶段采用“动态请求横屏窗口 + 横屏世界兜底 + 首次提醒”的方案。

弹射鸡蛋页内部做：

- 进入页面时请求 `window.Orientation.LANDSCAPE`。
- 退出页面时恢复 `window.Orientation.UNSPECIFIED`。
- `isLandscape = pageWidth > pageHeight` 作为布局判断。
- 横屏：显示完整游戏场景。
- 首次竖屏进入：显示一次横屏提醒。
- 游戏开始后：即使设备转回竖屏，也不盖遮罩、不暂停、不重置，游戏继续按横屏世界运行。

推荐文案：

```text
请横屏开始游戏
```

这样不会把大厅、扫雷和其他小游戏全局写死成横屏；横屏请求只在弹射鸡蛋页面生命周期内生效。

游戏世界坐标和 UI 排布只按横屏设计。竖屏不是另一套布局，只是用户设备方向变化后的显示状态。

## 横屏布局目标

横屏下建议布局：

```text
左侧：弹弓、主角鸡、剩余鸡蛋
中间：鸡蛋飞行轨迹、星星
右侧：目标、障碍、小木桶
顶部：关卡、分数、暂停
底部：木板平台和稻草地面
```

比例建议：

- 弹弓区：画面宽度 20% 到 28%。
- 目标区：画面宽度 35% 到 45%。
- 飞行区：画面中部保留完整抛物线。
- 地面高度：画面高度 18% 到 24%。

## 竖屏处理

竖屏不做完整玩法适配，也不做黑屏遮罩。

竖屏时可以：

- 首次进入时显示一次轻提示。
- 如果已经进入游戏，继续显示横屏世界。
- 不暂停物理 tick。
- 不禁用拖拽发射。
- 不重置关卡。

这样不会打断用户。代价是竖屏下可视空间可能不理想，但我们接受这个体验，因为弹射鸡蛋本身就是横屏游戏。

## 是否强制横屏

这里分两层：

- 页面级强制横屏布局：第一版就做，弹射鸡蛋只按横屏排布。
- 系统级窗口横屏方向：第一版已接入，进入弹射鸡蛋时动态请求主窗口横屏，退出时恢复默认。

原因：

- 当前是小游戏合集，不是弹射鸡蛋单独 App。
- 大厅、扫雷等页面更适合竖屏。
- 全局横屏会影响其他页面体验。
- 所以不在 `module.json5` 或 `EntryAbility` 里写死横屏，而是在 `EggSlingPage.ets` 生命周期里动态请求。

当前实现保留页面级横屏世界作为兜底。如果某些设备或模拟器不支持动态窗口方向请求，游戏内容仍然按横屏世界显示。

后续如果确认弹射鸡蛋要作为主推游戏，可以再评估：

- 是否给弹射鸡蛋单独 Ability。
- 是否从固定 `LANDSCAPE` 改成 `AUTO_ROTATION_LANDSCAPE`，让左右横拿方向跟随传感器。
- 是否大厅明确恢复竖屏，而不是恢复系统默认方向。

## 第一版开发任务

1. 在 `EggSlingPage.ets` 增加 `isLandscape()`。
2. 进入弹射鸡蛋时请求窗口横屏。
3. 退出弹射鸡蛋时恢复窗口默认方向。
4. 横屏时渲染完整游戏。
5. 首次竖屏进入时显示横屏提醒。
6. 游戏开始后转回竖屏也继续运行，不遮罩、不暂停。
7. 横屏布局重算弹弓、地面、目标、障碍坐标。
8. 让 Canvas 使用横屏世界尺寸做场景坐标。
9. 暂停、重开、返回大厅要在横屏布局下可点。

## 验收标准

- 竖屏首次进入弹射鸡蛋时，显示横屏提醒。
- 横屏进入弹射鸡蛋时，不显示横屏提醒。
- 进入弹射鸡蛋后，系统窗口和状态栏应切到横屏。
- 返回大厅后，系统窗口方向恢复默认，不影响其他小游戏。
- 横屏后游戏场景完整展开。
- 游戏中转回竖屏，不遮罩、不暂停、不重置。
- 弹弓在左侧，目标在右侧，中间有足够飞行空间。
- 拖拽发射不被系统手势区和顶部 HUD 干扰。

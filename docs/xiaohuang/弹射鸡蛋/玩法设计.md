# 弹射鸡蛋设计目标

## 定位

弹射鸡蛋是最适合强化《鸡窝游戏》品牌原创感的游戏，建议作为后续主推游戏重点打磨。

核心气质：

- 轻松、暖色、鸡窝后院感。
- 操作直观，拖拽蓄力后松手发射。
- 反馈夸张但不暴力，强调鸡蛋、稻草、木板和小鸡表情。
- 关卡节奏短，每局可以快速重试。

## 主题方向

玩法主题是：

鸡仔用弹弓把鸡蛋弹出去，打碎目标、收集星星、撞飞障碍。

玩家目标可以分层：

- 基础目标：击中或打碎指定目标。
- 进阶目标：尽量少用鸡蛋。
- 收集目标：沿途收集星星。
- 关卡挑战：利用不同鸡蛋类型解决障碍。

## 画面设计

场景像鸡窝后院。

主要视觉元素：

- 木板平台。
- 稻草堆。
- 鸡蛋弹弓。
- 小木桶。
- 软软的云。
- 暖色天空。

整体画面应该比扫雷更有主推游戏的表现力。扫雷偏棋盘和格子，弹射鸡蛋可以承担更多角色表情、物理反馈和场景动画。

## 鸡蛋类型

鸡蛋可以分类型，用颜色和行为区分。

第一批设计：

- 普通蛋：白色，基础飞行和碰撞。
- 弹弹蛋：粉色，碰撞后反弹更强。
- 爆裂蛋：黄色带裂纹，命中后造成范围破坏。
- 重量蛋：棕色，飞得更沉，适合撞开重障碍。
- 分裂蛋：带两个小点点，飞行中或命中后分裂成多个小蛋。

第一版实现不一定全部做完，但视觉和代码结构要预留蛋类型。

## 主角互动

发射前：

- 主角鸡站在弹弓旁边。
- 表情一脸淡定，半眯眼。

蓄力时：

- 眼睛稍微睁大。
- 身体可以轻微后仰或紧张。
- 弹弓皮筋拉开，鸡蛋跟随手指。

发射后：

- 主角鸡恢复半眯眼。
- 可以短暂有一个轻微眨眼或点头反馈。

这些表情不用一开始就做复杂动画，但状态应该在设计和代码里有明确分层：`idle`、`aiming`、`released`。

## 胜负文案

关卡胜利：

```text
蛋无虚发！
```

失败：

```text
这蛋白扔了。
```

这两个文案适合游戏气质，后续结算弹窗、Toast 或关卡结果页统一使用。

## 玩法方向

核心操作：

1. 玩家按住鸡蛋。
2. 向后拖拽蓄力和瞄准。
3. 松手发射。
4. 鸡蛋按抛物线飞出。
5. 鸡蛋和目标、星星、障碍发生碰撞。
6. 判断是否达成关卡目标。

第一版建议优先保证：

- 拖拽手感稳定。
- 发射角度和力度可预期。
- 鸡蛋飞行轨迹清楚。
- 命中目标有明确反馈。
- 胜利/失败条件简单可靠。

## 关卡目标

可以从简单到复杂逐步扩展：

- 打碎所有目标。
- 收集指定数量星星。
- 在限定鸡蛋数量内完成。
- 用指定蛋种完成目标。

当前阶段建议不要一开始做太多目标类型。先做“打碎目标 + 鸡蛋数量限制”，星星作为加分或额外收集。

## 素材规划

建议为弹射鸡蛋单独建立资源目录：

```text
entry/src/main/resources/rawfile/games/eggSling/
```

不要和扫雷素材混放。

建议素材分组：

- `background_*.png`：暖色天空、后院背景。
- `platform_*.png`：木板平台、稻草地面。
- `sling_*.png`：弹弓本体、皮筋状态。
- `egg_normal.png`
- `egg_bouncy.png`
- `egg_burst.png`
- `egg_heavy.png`
- `egg_split.png`
- `chick_idle.png`
- `chick_aiming.png`
- `chick_released.png`
- `target_*.png`
- `star_*.png`
- `obstacle_*.png`
- `popup_success_*.png`
- `popup_failure_*.png`

## 代码拆分建议

当前弹射鸡蛋主要集中在：

```text
entry/src/main/ets/games/eggSling/EggSlingPage.ets
```

后续建议拆分：

- `EggSlingPage.ets`：页面容器、生命周期、组件组装。
- `EggSlingTypes.ets`：鸡蛋、目标、障碍、关卡、碰撞体类型。
- `EggSlingLevels.ets`：关卡配置。
- `EggSlingPhysics.ets`：发射速度、重力、反弹、碰撞响应。
- `EggSlingRules.ets`：胜负判断、得分、剩余鸡蛋奖励。
- `EggSlingScene.ets`：场景绘制和主要交互区域。
- `EggSlingHud.ets`：顶部信息、剩余鸡蛋、关卡、分数。
- `EggSlingResultOverlay.ets`：胜利/失败弹窗。

优先拆纯逻辑，再拆 UI。这样后面可以给物理和胜负规则补测试。

## 后续讨论点

还需要继续确认：

- 第一版是否只保留普通蛋，还是先做普通蛋 + 弹弹蛋。
- 目标是“打碎木桶/靶子”，还是“撞飞障碍物”。
- 星星是必须收集，还是只作为加分。
- 是否保留当前代码里的关卡和得分方式。
- 鸡蛋数量默认几枚。
- 是否需要轨迹虚线辅助瞄准。
- 弹弓是否用 Canvas 绘制，还是用切图加 ArkUI 叠层。
- 第一批素材先画哪些，哪些可以用代码临时绘制。

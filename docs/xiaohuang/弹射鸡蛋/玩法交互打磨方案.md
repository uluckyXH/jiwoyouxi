# 弹射鸡蛋玩法交互打磨方案

## 当前判断

弹射鸡蛋现在已经有最小可玩原型：

- 按住鸡蛋。
- 拖拽蓄力。
- 松手发射。
- 鸡蛋受重力影响飞行。
- 鸡蛋命中目标或障碍后结算。
- 鸡蛋用完失败，目标清空过关。

但它还不是一个成熟游戏。当前问题主要不在素材，而在交互和物理反馈：

- 发射后缺少边界感，鸡蛋可以飞出画面。
- 拖拽力度、角度、轨迹反馈还不够可预期。
- 碰撞反馈偏弱，撞到障碍和撞到目标的差异不明显。
- 障碍只是静态矩形，没有“被撞动、被打碎、被推倒”的层次。
- 关卡还没有固定设计，随机生成不利于打磨手感。

## 先后顺序

当前阶段先不急着做正式关卡选择。

原因：

- 关卡选择页依赖关卡配置，而关卡配置又依赖核心手感。
- 如果发射、碰撞、边界和反馈还没调好，先做关卡只会产生一批不好玩的关卡。
- 素材后续会补，现阶段更应该用代码占位元素验证玩法。

第一阶段优先级：

1. 让拖拽发射稳定、可控。
2. 让鸡蛋飞行有边界、有反弹、有衰减。
3. 让命中、撞墙、落地都有明确反馈。
4. 让目标和障碍的行为差异先成立。
5. 再做固定关卡配置。
6. 最后再做正式关卡选择。

## 是否能用当前技术做好

可以做出轻量版“愤怒的小鸟式”玩法，但要明确边界。

当前技术栈：

```text
ArkTS + ArkUI + Canvas + 手写轻量物理
```

适合做：

- 拖拽蓄力。
- 抛物线飞行。
- 圆形和矩形碰撞。
- 边界反弹。
- 目标击碎。
- 星星收集。
- 简单障碍耐久。
- 固定关卡。
- 轻量粒子和命中反馈。

不适合一开始就做：

- 完整刚体物理堆叠。
- 多块木板真实倒塌。
- 复杂旋转、摩擦、关节约束。
- 大量动态物体同时碰撞。

所以第一版不要追求完整物理引擎效果。我们做“手感像、反馈像、规则好玩”的轻量版本。等玩法证明值得继续，再评估是否接入更专业的物理方案。

## 代码拆分方向

按项目规范，弹射鸡蛋不能继续把所有逻辑堆在 `EggSlingPage.ets`。

建议逐步拆成：

```text
entry/src/main/ets/games/eggSling/
  EggSlingPage.ets
  EggSlingTypes.ets
  EggSlingPhysics.ets
  EggSlingRules.ets
  EggSlingRenderer.ets
  EggSlingHud.ets
  EggSlingLevels.ets
```

第一步只拆最小必要文件：

- `EggSlingTypes.ets`：鸡蛋、目标、障碍、世界边界等类型。
- `EggSlingPhysics.ets`：重力、发射速度、反弹、边界碰撞参数。

先不拆 HUD 和 Renderer，避免一次改动太大。后续稳定后再拆。

## 当前交互打磨清单

### 1. 边界感

鸡蛋不能没有反馈地飞出画面。

第一版边界规则：

- 左边界：碰到后反弹，速度衰减。
- 右边界：碰到后反弹，速度衰减。
- 顶部边界：碰到后下弹，速度衰减。
- 地面：保留弹跳，但弹跳会快速衰减。
- 连续低速后认为鸡蛋停止。

边界表现先用代码画：

- 画面左右可以有很淡的木桩/边界线。
- 顶部边界不需要明显画出来，避免压迫感。
- 鸡蛋撞边界时可以轻微缩放或闪一下，后续再加粒子。

### 2. 拖拽手感

拖拽应该让玩家清楚知道：

- 现在拉了多大力度。
- 大概会往哪里飞。
- 超过最大拉力后不会继续变强。

第一版规则：

- 最大拖拽距离固定上限。
- 超过上限时，鸡蛋位置被限制在圆形范围内。
- 松手速度由拖拽向量计算。
- 后续补虚线轨迹预览。

### 3. 目标和障碍反馈

目标和障碍不能只是“消失或扣血”。

第一版规则：

- 目标被命中后消失并加分。
- 木板障碍有耐久，被撞后扣血。
- 障碍扣血后颜色变浅或裂纹占位。
- 后续素材到位后换成破碎帧或碎片。

### 4. 星星和额外目标

星星先不做为硬通关条件。

第一版规则：

- 星星是加分项。
- 收集数量影响后续 1 到 3 星结算。
- 关卡胜利仍以清空目标为主。

### 5. 关卡和关卡选择

先不做正式关卡选择页。

顺序：

1. 手感打磨。
2. 固定关卡配置。
3. 临时上一关/下一关调试按钮。
4. 正式关卡选择 UI。

正式关卡选择需要包含：

- 关卡编号。
- 是否通关。
- 最高星级。
- 是否锁定。

但这属于第二阶段。

## 素材预留

当前可以全部用代码占位：

- 鸡蛋：圆形或椭圆。
- 弹弓：线条和木杆。
- 目标：圆形或靶子图形。
- 障碍：矩形木板。
- 星星：Canvas 绘制五角星。
- 背景：渐变色、地面色块、简单云朵。

后续素材替换时，不改玩法规则，只替换渲染函数：

```text
drawEgg
drawSling
drawTarget
drawObstacle
drawStar
drawBackground
```

每个实体类型后续都应该有 `assetKey`，用于映射资源文件。

## 验收标准

第一阶段交互验收：

- 鸡蛋不会无反馈飞出画面。
- 鸡蛋撞左右边界和顶部边界有反弹。
- 鸡蛋落地后能自然衰减并停止。
- 拖拽距离有上限，不会无限蓄力。
- 发射方向和力度符合直觉。
- 撞目标、撞障碍、撞边界的反馈能区分。
- 返回大厅后窗口方向恢复正常。

完成这些后，再进入固定关卡配置。

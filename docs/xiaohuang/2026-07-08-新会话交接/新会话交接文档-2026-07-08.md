### 牌面渲染

用户最开始问过空当接龙里的牌是代码生成还是图片。

当前结论：

- 普通牌面不是 52 张牌图片。
- 牌面、数字、花色主要由 ArkUI 代码绘制。
- 背景、按钮、面板、空槽等 UI 装饰使用 rawfile 图片素材。

已修复的问题：

- 移动牌后，其他牌看起来没有花色/符号。
- 原因是牌面在紧凑覆盖状态和展开状态之间切换时，`ForEach` key 没把 covered/open 状态纳入，导致节点复用后显示状态不对。
- 解决方式是在 `FreecellBoard.ets` 的 key 中纳入状态，让移动后牌面正确刷新。

当前牌面逻辑：

- 被覆盖/压住的牌显示紧凑的 `rank + suit`。
- 展开的完整牌显示角标和中间花色。
- 移除了中间 J/Q/K 文字，避免和花色重复。
- 中央花色位置做过下移/上移微调，避免花色靠下或被裁切。

### 1. 玩法提示弹窗重新居中

用户截图对比：

- “玩法提示”弹窗偏移。

根因：

- `FreecellGuideOverlay` 之前使用了更窄的面板宽度：`Math.min(pageWidth - 88, 288)`。
- 同时还额外加了整体横向位移：

```ts
.translate({ x: this.panelOffsetX(), y: 0 })
```

- `FreecellExitConfirmOverlay` 没有这个横向位移，并且面板宽度是 `Math.min(pageWidth - 32, 318)`，所以视觉上正常。

最新未提交修复：

文件：

- `entry/src/main/ets/games/freecell/FreecellPromptOverlay.ets`

关键变化：

```ts
private panelWidth(): number {
  return Math.min(this.pageWidth - 32, 318);
}

private guideBodyWidth(): number {
  return Math.min(this.panelWidth() - 64, 226);
}
```

并且：

- 删除了玩法提示弹窗整体 `.translate(...)`。
- 面板宽度规则改为和“离开本局？”一致。
- 标题继续居中。
- 按钮继续居中。
- 正文区域用 `guideBodyWidth()` 单独约束宽度并居中。
- 正文内部仍保持左对齐，避免规则列表看起来松散。

用户最新反馈是“终于正常多了”，说明这个方向是对的。

## 后续接手建议

### 玩法提示弹窗

如果用户继续反馈偏移，优先看：

- `FreecellPromptOverlay.ets` 的 `panelWidth()`。
- `guideBodyWidth()`。
- `Column` 的 `.padding({ left: 28, right: 28, top: 56, bottom: 28 })`。

不要再用整体 `.translate({ x: ... })` 调玩法提示弹窗，这会再次造成标题、内容、按钮一起偏移。

正确策略：

- 面板整体交给 `CoopModalTransition` 的 `Alignment.Center` 居中。
- 标题和按钮保持居中。
- 内容区域单独设置宽度。
- 规则列表内部左对齐。

### HUD 统计块

如果继续调 HUD 数字底色，优先改：

```ts
FreecellTheme.hudValueBg
FreecellTheme.hudValueBorder
FreecellTheme.hudValueShadow
```

不要直接在 `FreecellPage.ets` 写硬编码颜色。

当前用户指定色值：

```text
#ffefce
```

### 牌面花色

如果用户继续反馈牌面花色位置、裁切、奇怪边缘，优先看：

- `FreecellBoard.ets` 里完整牌和 covered 牌的 Builder。
- 中央花色的尺寸和 position。
- 牌面 `.clip(true)`、`.borderRadius(...)`、`.backgroundColor(...)`。

当前牌面是代码渲染，不要去找 52 张牌面图片。

### 首次进入闪烁

如果用户继续反馈进入页面闪一下：

- 先确认是否是 `layoutReady` 之前背景 fallback 和真实背景图切换造成。
- 可考虑给背景图加载前后保持同色 overlay。
- 不建议恢复默认布局立即渲染，这会重新引入先按 `390x760` 渲染再重排的问题。

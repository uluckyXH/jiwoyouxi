# 游戏协作看板

更新时间：2026-06-09

这个页面用于多人协作时分配小游戏、记录开发状态和统一验收口径。具体实现步骤见 [新增小游戏接入指南.md](新增小游戏接入指南.md)，真机检查见 [真机验收清单.md](真机验收清单.md)。

## 状态定义

- `已接入`：首页可进入，有基础玩法和计分。
- `重点打磨`：当前主线优先优化，改动前先同步。
- `待认领`：可以新建 issue 或分支开始做。
- `设计中`：先定玩法、输入方式和得分规则，再开工。

## 认领规则

1. 开 issue，标题使用 `新增游戏：<游戏名>`。
2. 在 issue 里写清玩法、输入、计分、暂停/重开、失败条件。
3. 从 `main` 新建 `feature/<game-id>` 分支。
4. 按 [新增小游戏接入指南.md](新增小游戏接入指南.md) 接入页面和注册表。
5. 提交前跑 `./scripts/build-app.sh --stacktrace`。
6. 合并前按 [真机验收清单.md](真机验收清单.md) 做真机验证，并在 PR 里写清结果。

## 新游戏最小交付标准

- 首页能进入，游戏页能回大厅。
- 首屏就是可玩状态，不做说明落地页。
- 有暂停、重开、结算状态。
- 有非负分数，并能更新最高分。
- 无网络依赖，不卡死，不崩溃。
- 小屏不遮挡系统手势条，不出现明显文字重叠。

## 俄罗斯方块状态

已完成：

- 俄罗斯方块已按鸿蒙原生规范拆分：
  - `TetrisPage.ets`：页面组装、生命周期、状态机和计时调度。
  - `TetrisTypes.ets` / `TetrisRules.ets`：类型、常量和纯规则函数。
  - `TetrisBoard.ets`：Canvas 棋盘绘制，替代旧的 `Grid + 200 GridItem`。
  - `TetrisHud.ets` / `TetrisPreview.ets` / `TetrisControl.ets` / `TetrisOverlay.ets`：HUD、预览、控制和弹窗。
- 7-bag 随机队列、3 个 next 预览、幽灵落点、消行计分、等级、锁定延迟已迁到规则层和状态机。
- 页面 UI 保持奶油玻璃 / HarmonyOS 原生质感：
  - 顶部返回/标题/最高分/暂停
  - 深色棋盘和奶油外框
  - 左右状态卡
  - 底部 4 键控制面板
- 已移除底部逆时针旋转按钮和独立硬降按钮。
- 暂存 Hold 已恢复为可操作入口，点按左侧“暂存”卡片触发，每个方块限用一次。
- 规则回归测试已新增：`node --no-warnings scripts/test-tetris-rules.mjs`。

待复测：

- 控制输入仍使用控制面板统一 `onTouch` 分发：
  - 点按 `←/→/↓/↻`
  - 左右长按 180ms 后每 80ms 连续移动
  - `↓` 长按 350ms 硬降
- Canvas 棋盘真机渲染清晰度、尺寸适配和触摸手感。
- 暂存 Hold 真机点按命中和状态提示。
- 暂停/继续、结算、最高分回写。

已知风险：

- 早期版本中，底部子控件单独 `onClick/onTouch/gesture` 在真机 `uiInput` 下有事件不触发问题，所以改为父面板统一触摸分发。
- 本轮已完成构建和规则测试，但还缺真机试玩验收；如果 Canvas 在特定机型上尺寸或清晰度异常，优先调 `TetrisBoard.ets` 和 `TetrisPage.ets` 的 `cellSize()`。

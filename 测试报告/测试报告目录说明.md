# 测试报告目录

本目录用于保留项目测试报告。每次新增游戏、重构游戏、调整全局转场、修改存储或修复关键缺陷后，都应新增一份 Markdown 测试报告。

## 命名

```text
YYYY-MM-DD-<测试范围>.md
```

示例：

```text
2026-06-10-游戏切换动画.md
2026-06-10-扫雷规则.md
```

## 留档规则

- Markdown 测试报告进入 Git。
- 大型截图、录屏、layout dump 和临时日志默认放本机 `phone-shots/`，不进入 Git。
- 如果必须提交关键截图，先压缩并在报告中说明原因。
- 文档-only 改动可以不新增测试报告，但提交说明或 PR 描述中要写明未运行构建的原因。
